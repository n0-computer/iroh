# Iroh Internals Documentation

This directory contains documentation about iroh's internal architecture, state machines,
and component lifecycles. It is designed to be **source-first**: most content can be
regenerated by an LLM reading the current source code.

## How This Documentation Works

Each document contains three kinds of content:

### 1. Human-written prose (stable)
Explanatory text that provides context, design rationale, and high-level understanding.
This is written once and updated infrequently by humans.

### 2. Auto-generated sections (regenerable)
Sections between `<!-- BEGIN GENERATED -->` and `<!-- END GENERATED -->` markers.
These contain Mermaid diagrams and structured descriptions derived from the source code.
They can be regenerated at any time by an LLM.

Each generated section includes an HTML comment with:
- **Source**: the file(s) the LLM should read
- **Prompt**: what to extract and how to render it

### 3. Regeneration instructions
The `_prompts/` directory contains shared instructions for the LLM regeneration process.

## Regenerating

To regenerate a specific section, provide the LLM with:
1. The markdown file containing the section
2. The source file(s) referenced in the section's HTML comment
3. The prompt from `_prompts/regenerate.md`

The LLM should replace only the content between the `BEGIN GENERATED` and `END GENERATED`
markers, preserving all human-written content.

## Viewing Mermaid Diagrams

Mermaid diagrams render natively on GitHub. For local viewing, use:
- VS Code with the "Markdown Preview Mermaid Support" extension
- `mmdc` CLI from `@mermaid-js/mermaid-cli` to render to SVG
