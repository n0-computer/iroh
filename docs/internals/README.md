# Iroh Internals Documentation

Source-first documentation of iroh's internal architecture. Most content can be
regenerated by an LLM reading the current source code.

| Document | What it covers |

| [endpoint-shutdown.md](endpoint-shutdown.md) | Graceful shutdown sequence |

## How This Works

Each document has three kinds of content:

**Human-written prose** — design rationale, "why" explanations. Updated by humans.

**Generated sections** — SVG diagrams and structured descriptions between
`<!-- BEGIN GENERATED SECTION -->` and `<!-- END GENERATED SECTION -->` markers.
Each marker includes `Source:` (files to read) and `Prompt:` (what to extract).
An LLM regenerates these by reading the source and following
[_prompts/regenerate.md](_prompts/regenerate.md).

**SVG diagrams** — in `diagrams/`, generated directly as XML by the LLM. No external
tools required. Dark background, iroh brand colors. Validated with `xmllint`.

## Validating

```sh
./docs/internals/validate.sh
```

Checks all SVGs are well-formed XML. Requires `xmllint` (ships with macOS, `libxml2` on Linux).

## Regenerating a section

Give the LLM:
1. The markdown file with the section to regenerate
2. The source file(s) from the `Source:` field
3. The style guide from `_prompts/regenerate.md`

The LLM replaces content between markers, regenerates SVG files, and leaves human prose untouched.
