1//! Node address discovery.
2//!
3//! To connect to an iroh node a [`NodeAddr`] is needed, which may contain a
4//! [`RelayUrl`] or one or more *direct addresses* in addition to the [`NodeId`].
5//!
6//! Since there is a conversion from [`NodeId`] to [`NodeAddr`], you can also use
7//! connect directly with a [`NodeId`].
8//!
9//! For this to work however, the endpoint has to get the addressing  information by
10//! other means.  This can be done by manually calling [`Endpoint::add_node_addr`],
11//! but that still requires knowing the other addressing information.
12//!
13//! Node discovery is an automated system for an [`Endpoint`] to retrieve this addressing
14//! information.  Each iroh node will automatically publish their own addressing
15//! information.  Usually this means publishing which [`RelayUrl`] to use for their
16//! [`NodeId`], but they could also publish their direct addresses.
17//!
18//! The [`Discovery`] trait is used to define node discovery.  This allows multiple
19//! implementations to co-exist because there are many possible ways to implement this.
20//! Each [`Endpoint`] can use the discovery mechanisms most suitable to the application.
21//! The [`Builder::add_discovery`] method is used to add a discovery mechanism to an
22//! [`Endpoint`].
23//!
24//! Some generally useful discovery implementations are provided:
25//!
26//! - [`StaticProvider`] which allows application to add and remove out-of-band addressing
27//!   information.
28//!
29//! - The [`DnsDiscovery`] which performs lookups via the standard DNS systems.  To publish
30//!   to this DNS server a [`PkarrPublisher`] is needed.  [Number 0] runs a public instance
31//!   of a [`PkarrPublisher`] with attached DNS server which is globally available and a
32//!   reliable default choice.
33//!
34//! - The [`PkarrResolver`] which can perform lookups from designated [pkarr relay servers]
35//!   using HTTP.
36//!
37//! - [`MdnsDiscovery`]: mdns::MdnsDiscovery which uses the crate `swarm-discovery`, an
38//!   opinionated mDNS implementation, to discover nodes on the local network.
39//!
40//! - The [`DhtDiscovery`] also uses the [`pkarr`] system but can also publish and lookup
41//!   records to/from the Mainline DHT.
42//!
43//! To use multiple discovery systems simultaneously you can call [`Builder::add_discovery`].
44//! This will use [`ConcurrentDiscovery`] under the hood, which performs lookups to all
45//! discovery systems at the same time.
46//!
47//! [`Builder::add_discovery`] takes any type that implements [`IntoDiscovery`]. You can
48//! implement that trait on a builder struct if your discovery service needs information
49//! from the endpoint it is mounted on. During endpoint construction, your discovery service
50//! is built by calling [`IntoDiscovery::into_discovery`], passing a [`DiscoveryContext`] to your
51//! builder. The [`DiscoveryContext`] gives access to the endpoint's secret key and DNS resolver.
52//!
53//! If your discovery service does not need any information from its endpoint, you can
54//! pass the discovery service directly to [`Builder::add_discovery`]: All types that
55//! implement [`Discovery`] also have a blanket implementation of [`IntoDiscovery`].
56//!
57//! # Examples
58//!
59//! A very common setup is to enable DNS discovery, which needs to be done in two parts as a
60//! [`PkarrPublisher`] and [`DnsDiscovery`]:
61//!
62//! ```no_run
63//! use iroh::{
64//!     discovery::{dns::DnsDiscovery, pkarr::PkarrPublisher},
65//!     Endpoint, SecretKey,
66//! };
67//!
68//! # async fn wrapper() -> n0_snafu::Result<()> {
69//! let ep = Endpoint::builder()
70//!     .add_discovery(PkarrPublisher::n0_dns())
71//!     .add_discovery(DnsDiscovery::n0_dns())
72//!     .bind()
73//!     .await?;
74//! # Ok(())
75//! # }
76//! ```
77//!
78//! To also enable [`MdnsDiscovery`] it can be added as another service.
79//!
80//! ```no_run
81//! #[cfg(feature = "discovery-local-network")]
82//! # {
83//! # use iroh::{
84//! #    discovery::{dns::DnsDiscovery, pkarr::PkarrPublisher, mdns::MdnsDiscovery},
85//! #    Endpoint, SecretKey,
86//! # };
87//! #
88//! # async fn wrapper() -> n0_snafu::Result<()> {
89//! let ep = Endpoint::builder()
90//!     .add_discovery(PkarrPublisher::n0_dns())
91//!     .add_discovery(DnsDiscovery::n0_dns())
92//!     .add_discovery(MdnsDiscovery::builder())
93//!     .bind()
94//!     .await?;
95//! # Ok(())
96//! # }
97//! # }
98//! ```
99//!
100//! [`NodeAddr`]: iroh_base::NodeAddr
101//! [`RelayUrl`]: crate::RelayUrl
102//! [`Builder::add_discovery`]: crate::endpoint::Builder::add_discovery
103//! [`DnsDiscovery`]: dns::DnsDiscovery
104//! [Number 0]: https://n0.computer
105//! [`PkarrResolver`]: pkarr::PkarrResolver
106//! [`PkarrPublisher`]: pkarr::PkarrPublisher
107//! [`DhtDiscovery`]: pkarr::dht::DhtDiscovery
108//! [pkarr relay servers]: https://pkarr.org/#servers
109//! [`MdnsDiscovery`]: mdns::MdnsDiscovery
110//! [`StaticProvider`]: static_provider::StaticProvider

140/// Trait for structs that can be converted into [`Discovery`].
141///
142/// This trait is implemented on builders for discovery services. Any type that implements this
143/// trait can be added as a discovery service in [`Builder::add_discovery`].
144///
145/// Any type that implements [`Discovery`] also implements [`IntoDiscovery`].
146///
147/// Iroh uses this trait to allow configuring the set of discovery services on the endpoint
148/// builder, while providing the discovery services access to information about the endpoint
149/// creation via the [`DiscoveryContext`] parameter to [`IntoDiscovery::into_discovery`].
150///
151/// [`Builder::add_discovery`]: crate::endpoint::Builder::add_discovery
152pub trait IntoDiscovery: Send + Sync + std::fmt::Debug + 'static {
153    /// Turns this discovery builder into a ready-to-use discovery service.
154    ///
155    /// The [`DiscoveryContext`] contains information about the [`Endpoint`] onto which this
156    /// discovery service is being added. It can be used by discovery services that need
157    /// a DNS resolver, or the endpoint's secret key to sign messages.
158    ///
159    /// If an error is returned, building the endpoint will fail with this error.
160    fn into_discovery(
161        self,
162        context: &DiscoveryContext,
163    ) -> Result<impl Discovery, IntoDiscoveryError>;
164}
165
166/// Blanket no-op impl of `IntoDiscovery` for `T: Discovery`.
167impl<T: Discovery> IntoDiscovery for T {
168    fn into_discovery(
169        self,
170        _context: &DiscoveryContext,
171    ) -> Result<impl Discovery, IntoDiscoveryError> {
172        Ok(self)
173    }
174}

296/// Node discovery for [`super::Endpoint`].
297///
298/// This trait defines publishing and resolving addressing information for a [`NodeId`].
299/// This enables connecting to other nodes with only knowing the [`NodeId`], by using this
300/// [`Discovery`] system to look up the actual addressing information.  It is common for
301/// implementations to require each node to publish their own information before it can be
302/// looked up by other nodes.
303///
304/// The published addressing information can include both a [`RelayUrl`] and/or direct
305/// addresses. See [`NodeData`] for details.
306///
307/// To allow for discovery, the [`super::Endpoint`] will call `publish` whenever
308/// discovery information changes. If a discovery mechanism requires a periodic
309/// refresh, it should start its own task.
310///
311/// [`RelayUrl`]: crate::RelayUrl
312pub trait Discovery: std::fmt::Debug + Send + Sync + 'static {
313    /// Publishes the given [`NodeData`] to the discovery mechanism.
314    ///
315    /// This is fire and forget, since the [`Endpoint`] can not wait for successful
316    /// publishing. If publishing is async, the implementation should start it's own task.
317    ///
318    /// This will be called from a tokio task, so it is safe to spawn new tasks.
319    /// These tasks will be run on the runtime of the [`super::Endpoint`].
320    fn publish(&self, _data: &NodeData) {}
321
322    /// Resolves the [`DiscoveryItem`] for the given [`NodeId`].
323    ///
324    /// Once the returned [`BoxStream`] is dropped, the service should stop any pending
325    /// work.
326    fn resolve(
327        &self,
328        _node_id: NodeId,
329    ) -> Option<BoxStream<Result<DiscoveryItem, DiscoveryError>>> {
330        None
331    }
332
333    /// Subscribe to all addresses that get *passively* discovered.
334    ///
335    /// An implementation may choose to defer emitting passively discovered nodes
336    /// until the stream is actually polled. To avoid missing discovered nodes,
337    /// poll the stream as soon as possible.
338    ///
339    /// If you do not regularly poll the stream, you may miss discovered nodes.
340    ///
341    /// Any discovery systems that only discover when explicitly resolving a
342    /// specific [`NodeId`] do not need to implement this method. Any nodes or
343    /// addresses that are discovered by calling `resolve` should NOT be added
344    /// to the `subscribe` stream.
345    ///
346    /// Discovery systems that are capable of receiving information about [`NodeId`]s
347    /// and their addressing information without explicitly calling `resolve`, i.e.,
348    /// systems that do "passive" discovery, should implement this method. If
349    /// `subscribe` is called multiple times, the passively discovered addresses
350    /// should be sent on all streams.
351    ///
352    /// The [`crate::endpoint::Endpoint`] will `subscribe` to the discovery system
353    /// and add the discovered addresses to the internal address book as they arrive
354    /// on this stream.
355    fn subscribe(&self) -> Option<BoxStream<DiscoveryItem>> {
356        None
357    }
358}

362/// Node discovery results from [`Discovery`] services.
363///
364/// This is the item in the streams returned from [`Discovery::resolve`] and
365/// [`Discovery::subscribe`]. It contains the [`NodeData`] about the discovered node,
366/// and some additional metadata about the discovery.
367///
368/// This struct derefs to [`NodeData`], so you can access the methods from [`NodeData`]
369/// directly from [`DiscoveryItem`].
370#[derive(Debug, Clone)]
371pub struct DiscoveryItem {
372    /// The node info for the node, as discovered by the the discovery service.
373    node_info: NodeInfo,
374    /// A static string to identify the discovery source.
375    ///
376    /// Should be uniform per discovery service.
377    provenance: &'static str,
378    /// Optional timestamp when this node address info was last updated.
379    ///
380    /// Must be microseconds since the unix epoch.
381    // TODO(ramfox): this is currently unused. As we develop more `DiscoveryService`s, we may discover that we do not need this. It is only truly relevant when comparing `relay_urls`, since we can attempt to dial any number of socket addresses, but expect each node to have one "home relay" that we will attempt to contact them on. This means we would need some way to determine which relay url to choose between, if more than one relay url is reported.
382    last_updated: Option<u64>,
383}

449/// A discovery service that combines multiple discovery sources.
450///
451/// The discovery services will resolve concurrently.
452#[derive(Debug, Default)]
453pub struct ConcurrentDiscovery {
454    services: Vec<Box<dyn Discovery>>,
455}
456
457impl ConcurrentDiscovery {
458    /// Creates an empty [`ConcurrentDiscovery`].
459    pub fn empty() -> Self {
460        Self::default()
461    }
462
463    /// Creates a new [`ConcurrentDiscovery`].
464    pub fn from_services(services: Vec<Box<dyn Discovery>>) -> Self {
465        Self { services }
466    }
467
468    /// Adds a [`Discovery`] service.
469    pub fn add(&mut self, service: impl Discovery + 'static) {
470        self.services.push(Box::new(service));
471    }
472}
473
474impl<T> From<T> for ConcurrentDiscovery
475where
476    T: IntoIterator<Item = Box<dyn Discovery>>,
477{
478    fn from(iter: T) -> Self {
479        let services = iter.into_iter().collect::<Vec<_>>();
480        Self { services }
481    }
482}
483
484impl Discovery for ConcurrentDiscovery {
485    fn publish(&self, data: &NodeData) {
486        for service in &self.services {
487            service.publish(data);
488        }
489    }
490
491    fn resolve(&self, node_id: NodeId) -> Option<BoxStream<Result<DiscoveryItem, DiscoveryError>>> {
492        let streams = self
493            .services
494            .iter()
495            .filter_map(|service| service.resolve(node_id));
496
497        let streams = n0_future::MergeBounded::from_iter(streams);
498        Some(Box::pin(streams))
499    }
500
501    fn subscribe(&self) -> Option<BoxStream<DiscoveryItem>> {
502        let mut streams = vec![];
503        for service in self.services.iter() {
504            if let Some(stream) = service.subscribe() {
505                streams.push(stream)
506            }
507        }
508
509        let streams = n0_future::MergeBounded::from_iter(streams);
510        Some(Box::pin(streams))
511    }
512}

514/// Maximum duration since the last control or data message received from an endpoint to make us
515/// start a discovery task.
516const MAX_AGE: Duration = Duration::from_secs(10);
517
518/// A wrapper around a tokio task which runs a node discovery.
519pub(super) struct DiscoveryTask {
520    on_first_rx: oneshot::Receiver<Result<(), DiscoveryError>>,
521    _task: AbortOnDropHandle<()>,
522}
523
524impl DiscoveryTask {
525    /// Starts a discovery task.
526    pub(super) fn start(ep: Endpoint, node_id: NodeId) -> Result<Self, DiscoveryError> {
527        ensure!(ep.discovery().is_some(), NoServiceConfiguredSnafu);
528        let (on_first_tx, on_first_rx) = oneshot::channel();
529        let me = ep.node_id();
530        let task = task::spawn(
531            async move { Self::run(ep, node_id, on_first_tx).await }.instrument(
532                error_span!("discovery", me = %me.fmt_short(), node = %node_id.fmt_short()),
533            ),
534        );
535        Ok(Self {
536            _task: AbortOnDropHandle::new(task),
537            on_first_rx,
538        })
539    }
540
541    /// Starts a discovery task after a delay and only if no path to the node was recently active.
542    ///
543    /// This returns `None` if we received data or control messages from the remote endpoint
544    /// recently enough. If not it returns a [`DiscoveryTask`].
545    ///
546    /// If `delay` is set, the [`DiscoveryTask`] will first wait for `delay` and then check again
547    /// if we recently received messages from remote endpoint. If true, the task will abort.
548    /// Otherwise, or if no `delay` is set, the discovery will be started.
549    pub(super) fn maybe_start_after_delay(
550        ep: &Endpoint,
551        node_id: NodeId,
552        delay: Option<Duration>,
553    ) -> Result<Option<Self>, DiscoveryError> {
554        // If discovery is not needed, don't even spawn a task.
555        if !Self::needs_discovery(ep, node_id) {
556            return Ok(None);
557        }
558        ensure!(ep.discovery().is_some(), NoServiceConfiguredSnafu);
559        let (on_first_tx, on_first_rx) = oneshot::channel();
560        let ep = ep.clone();
561        let me = ep.node_id();
562        let task = task::spawn(
563            async move {
564                // If delay is set, wait and recheck if discovery is needed. If not, early-exit.
565                if let Some(delay) = delay {
566                    time::sleep(delay).await;
567                    if !Self::needs_discovery(&ep, node_id) {
568                        debug!("no discovery needed, abort");
569                        on_first_tx.send(Ok(())).ok();
570                        return;
571                    }
572                }
573                Self::run(ep, node_id, on_first_tx).await
574            }
575            .instrument(
576                error_span!("discovery", me = %me.fmt_short(), node = %node_id.fmt_short()),
577            ),
578        );
579        Ok(Some(Self {
580            _task: AbortOnDropHandle::new(task),
581            on_first_rx,
582        }))
583    }
584
585    /// Waits until the discovery task produced at least one result.
586    pub(super) async fn first_arrived(&mut self) -> Result<(), DiscoveryError> {
587        let fut = &mut self.on_first_rx;
588        fut.await.expect("sender dropped")?;
589        Ok(())
590    }
591
592    fn create_stream(
593        ep: &Endpoint,
594        node_id: NodeId,
595    ) -> Result<BoxStream<Result<DiscoveryItem, DiscoveryError>>, DiscoveryError> {
596        let discovery = ep.discovery().ok_or(NoServiceConfiguredSnafu.build())?;
597        let stream = discovery
598            .resolve(node_id)
599            .ok_or(NoResultsSnafu { node_id }.build())?;
600        Ok(stream)
601    }
602
603    /// We need discovery if we have no paths to the node, or if the paths we do have
604    /// have timed out.
605    fn needs_discovery(ep: &Endpoint, node_id: NodeId) -> bool {
606        match ep.remote_info(node_id) {
607            // No info means no path to node -> start discovery.
608            None => true,
609            Some(info) => {
610                match (
611                    info.last_received(),
612                    info.relay_url.as_ref().and_then(|r| r.last_alive),
613                ) {
614                    // No path to node -> start discovery.
615                    (None, None) => true,
616                    // If we haven't received on direct addresses or the relay for MAX_AGE,
617                    // start discovery.
618                    (Some(elapsed), Some(elapsed_relay)) => {
619                        elapsed > MAX_AGE && elapsed_relay > MAX_AGE
620                    }
621                    (Some(elapsed), _) | (_, Some(elapsed)) => elapsed > MAX_AGE,
622                }
623            }
624        }
625    }
626
627    async fn run(
628        ep: Endpoint,
629        node_id: NodeId,
630        on_first_tx: oneshot::Sender<Result<(), DiscoveryError>>,
631    ) {
632        let mut stream = match Self::create_stream(&ep, node_id) {
633            Ok(stream) => stream,
634            Err(err) => {
635                on_first_tx.send(Err(err)).ok();
636                return;
637            }
638        };
639        let mut on_first_tx = Some(on_first_tx);
640        debug!("starting");
641        loop {
642            match stream.next().await {
643                Some(Ok(r)) => {
644                    let provenance = r.provenance;
645                    let node_addr = r.to_node_addr();
646                    if node_addr.is_empty() {
647                        debug!(%provenance, "empty address found");
648                        continue;
649                    }
650                    debug!(%provenance, addr = ?node_addr, "new address found");
651                    ep.add_node_addr_with_source(node_addr, provenance).ok();
652                    if let Some(tx) = on_first_tx.take() {
653                        tx.send(Ok(())).ok();
654                    }
655                    // Send the discovery item to the subscribers of the discovery broadcast stream.
656                    ep.discovery_subscribers().send(r);
657                }
658                Some(Err(err)) => {
659                    warn!(?err, "discovery service produced error");
660                    break;
661                }
662                None => break,
663            }
664        }
665        if let Some(tx) = on_first_tx.take() {
666            tx.send(Err(NoResultsSnafu { node_id }.build())).ok();
667        }
668    }
669}

671/// Error returned when a discovery watch stream lagged too far behind.
672///
673/// The stream returned from [`Endpoint::discovery_stream`] yields this error
674/// if the loop in which the stream is processed cannot keep up with the emitted
675/// discovery events. Attempting to read the next item from the channel afterwards
676/// will return the oldest [`DiscoveryItem`] that is still retained.
677///
678/// Includes the number of skipped messages.
679#[derive(Debug, Snafu)]
680#[snafu(display("channel lagged by {val}"))]
681pub struct Lagged {
682    /// The number of skipped messages
683    pub val: u64,
684}
685
686#[derive(Clone, Debug)]
687pub(super) struct DiscoverySubscribers {
688    inner: tokio::sync::broadcast::Sender<DiscoveryItem>,
689}
690
691impl DiscoverySubscribers {
692    pub(crate) fn new() -> Self {
693        // TODO: Make capacity configurable from the endpoint builder?
694        // This is the maximum number of [`DiscoveryItem`]s held by the channel if
695        // subscribers are stalled.
696        const CAPACITY: usize = 128;
697        Self {
698            inner: tokio::sync::broadcast::Sender::new(CAPACITY),
699        }
700    }
701
702    pub(crate) fn subscribe(&self) -> impl Stream<Item = Result<DiscoveryItem, Lagged>> {
703        use tokio_stream::wrappers::{errors::BroadcastStreamRecvError, BroadcastStream};
704        let recv = self.inner.subscribe();
705        BroadcastStream::new(recv).map_err(|BroadcastStreamRecvError::Lagged(n)| Lagged { val: n })
706    }
707
708    pub(crate) fn send(&self, item: DiscoveryItem) {
709        // `broadcast::Sender::send` returns an error if the channel has no subscribers,
710        // which we don't care about.
711        self.inner.send(item).ok();
712    }
713}

1047    async fn new_endpoint(
1048        secret: SecretKey,
1049        disco: impl IntoDiscovery + 'static,
1050    ) -> (Endpoint, AbortOnDropHandle<Result<()>>) {
1051        let ep = Endpoint::builder()
1052            .secret_key(secret)
1053            .discovery(disco)
1054            .relay_mode(RelayMode::Disabled)
1055            .alpns(vec![TEST_ALPN.to_vec()])
1056            .bind()
1057            .await
1058            .unwrap();
1059
1060        let handle = tokio::spawn({
1061            let ep = ep.clone();
1062            async move {
1063                // Keep connections alive until the task is dropped.
1064                let mut connections = Vec::new();
1065                // we skip accept() errors, they can be caused by retransmits
1066                while let Some(connecting) = ep.accept().await.and_then(|inc| inc.accept().ok()) {
1067                    // Just accept incoming connections, but don't do anything with them.
1068                    let conn = connecting.await.context("connecting")?;
1069                    connections.push(conn);
1070                }
1071
1072                Ok::<_, Error>(())
1073            }
1074        });
1075
1076        (ep, AbortOnDropHandle::new(handle))
1077    }
