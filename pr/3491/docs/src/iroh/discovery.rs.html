1//! Node address discovery.
2//!
3//! To connect to an iroh node a [`NodeAddr`] is needed, which may contain a
4//! [`RelayUrl`] or one or more *direct addresses* in addition to the [`NodeId`].
5//!
6//! Since there is a conversion from [`NodeId`] to [`NodeAddr`], you can also use
7//! connect directly with a [`NodeId`].
8//!
9//! For this to work however, the endpoint has to get the addressing  information by
10//! other means.  This can be done by manually calling [`Endpoint::add_node_addr`],
11//! but that still requires knowing the other addressing information.
12//!
13//! Node discovery is an automated system for an [`Endpoint`] to retrieve this addressing
14//! information.  Each iroh node will automatically publish their own addressing
15//! information.  Usually this means publishing which [`RelayUrl`] to use for their
16//! [`NodeId`], but they could also publish their direct addresses.
17//!
18//! The [`Discovery`] trait is used to define node discovery.  This allows multiple
19//! implementations to co-exist because there are many possible ways to implement this.
20//! Each [`Endpoint`] can use the discovery mechanisms most suitable to the application.
21//! The [`Builder::add_discovery`] method is used to add a discovery mechanism to an
22//! [`Endpoint`].
23//!
24//! Some generally useful discovery implementations are provided:
25//!
26//! - [`StaticProvider`] which allows application to add and remove out-of-band addressing
27//!   information.
28//!
29//! - The [`DnsDiscovery`] which performs lookups via the standard DNS systems.  To publish
30//!   to this DNS server a [`PkarrPublisher`] is needed.  [Number 0] runs a public instance
31//!   of a [`PkarrPublisher`] with attached DNS server which is globally available and a
32//!   reliable default choice.
33//!
34//! - The [`PkarrResolver`] which can perform lookups from designated [pkarr relay servers]
35//!   using HTTP.
36//!
37//! - [`MdnsDiscovery`]: mdns::MdnsDiscovery which uses the crate `swarm-discovery`, an
38//!   opinionated mDNS implementation, to discover nodes on the local network.
39//!
40//! - The [`DhtDiscovery`] also uses the [`pkarr`] system but can also publish and lookup
41//!   records to/from the Mainline DHT.
42//!
43//! To use multiple discovery systems simultaneously you can call [`Builder::add_discovery`].
44//! This will use [`ConcurrentDiscovery`] under the hood, which performs lookups to all
45//! discovery systems at the same time.
46//!
47//! [`Builder::add_discovery`] takes any type that implements [`IntoDiscovery`]. You can
48//! implement that trait on a builder struct if your discovery service needs information
49//! from the endpoint it is mounted on. During endpoint construction, your discovery service
50//! is built by calling [`IntoDiscovery::into_discovery`], passing a [`DiscoveryContext`] to your
51//! builder. The [`DiscoveryContext`] gives access to the endpoint's secret key and DNS resolver.
52//!
53//! If your discovery service does not need any information from its endpoint, you can
54//! pass the discovery service directly to [`Builder::add_discovery`]: All types that
55//! implement [`Discovery`] also have a blanket implementation of [`IntoDiscovery`].
56//!
57//! # Examples
58//!
59//! A very common setup is to enable DNS discovery, which needs to be done in two parts as a
60//! [`PkarrPublisher`] and [`DnsDiscovery`]:
61//!
62//! ```no_run
63//! use iroh::{
64//!     Endpoint, SecretKey,
65//!     discovery::{dns::DnsDiscovery, pkarr::PkarrPublisher},
66//! };
67//!
68//! # async fn wrapper() -> n0_snafu::Result<()> {
69//! let ep = Endpoint::builder()
70//!     .add_discovery(PkarrPublisher::n0_dns())
71//!     .add_discovery(DnsDiscovery::n0_dns())
72//!     .bind()
73//!     .await?;
74//! # Ok(())
75//! # }
76//! ```
77//!
78//! To also enable [`MdnsDiscovery`] it can be added as another service.
79//!
80//! ```no_run
81//! #[cfg(feature = "discovery-local-network")]
82//! # {
83//! # use iroh::{
84//! #    discovery::{dns::DnsDiscovery, pkarr::PkarrPublisher, mdns::MdnsDiscovery},
85//! #    Endpoint, SecretKey,
86//! # };
87//! #
88//! # async fn wrapper() -> n0_snafu::Result<()> {
89//! let ep = Endpoint::builder()
90//!     .add_discovery(PkarrPublisher::n0_dns())
91//!     .add_discovery(DnsDiscovery::n0_dns())
92//!     .add_discovery(MdnsDiscovery::builder())
93//!     .bind()
94//!     .await?;
95//! # Ok(())
96//! # }
97//! # }
98//! ```
99//!
100//! [`NodeAddr`]: iroh_base::NodeAddr
101//! [`RelayUrl`]: crate::RelayUrl
102//! [`Builder::add_discovery`]: crate::endpoint::Builder::add_discovery
103//! [`DnsDiscovery`]: dns::DnsDiscovery
104//! [Number 0]: https://n0.computer
105//! [`PkarrResolver`]: pkarr::PkarrResolver
106//! [`PkarrPublisher`]: pkarr::PkarrPublisher
107//! [`DhtDiscovery`]: pkarr::dht::DhtDiscovery
108//! [pkarr relay servers]: https://pkarr.org/#servers
109//! [`MdnsDiscovery`]: mdns::MdnsDiscovery
110//! [`StaticProvider`]: static_provider::StaticProvider

140/// Trait for structs that can be converted into [`Discovery`].
141///
142/// This trait is implemented on builders for discovery services. Any type that implements this
143/// trait can be added as a discovery service in [`Builder::add_discovery`].
144///
145/// Any type that implements [`Discovery`] also implements [`IntoDiscovery`].
146///
147/// Iroh uses this trait to allow configuring the set of discovery services on the endpoint
148/// builder, while providing the discovery services access to information about the endpoint
149/// creation via the [`DiscoveryContext`] parameter to [`IntoDiscovery::into_discovery`].
150///
151/// [`Builder::add_discovery`]: crate::endpoint::Builder::add_discovery
152pub trait IntoDiscovery: Send + Sync + std::fmt::Debug + 'static {
153    /// Turns this discovery builder into a ready-to-use discovery service.
154    ///
155    /// The [`DiscoveryContext`] contains information about the [`Endpoint`] onto which this
156    /// discovery service is being added. It can be used by discovery services that need
157    /// a DNS resolver, or the endpoint's secret key to sign messages.
158    ///
159    /// If an error is returned, building the endpoint will fail with this error.
160    fn into_discovery(
161        self,
162        context: &DiscoveryContext,
163    ) -> Result<impl Discovery, IntoDiscoveryError>;
164}
165
166/// Blanket no-op impl of `IntoDiscovery` for `T: Discovery`.
167impl<T: Discovery> IntoDiscovery for T {
168    fn into_discovery(
169        self,
170        _context: &DiscoveryContext,
171    ) -> Result<impl Discovery, IntoDiscoveryError> {
172        Ok(self)
173    }
174}

296/// Node discovery for [`super::Endpoint`].
297///
298/// This trait defines publishing and resolving addressing information for a [`NodeId`].
299/// This enables connecting to other nodes with only knowing the [`NodeId`], by using this
300/// [`Discovery`] system to look up the actual addressing information.  It is common for
301/// implementations to require each node to publish their own information before it can be
302/// looked up by other nodes.
303///
304/// The published addressing information can include both a [`RelayUrl`] and/or direct
305/// addresses. See [`NodeData`] for details.
306///
307/// To allow for discovery, the [`super::Endpoint`] will call `publish` whenever
308/// discovery information changes. If a discovery mechanism requires a periodic
309/// refresh, it should start its own task.
310///
311/// [`RelayUrl`]: crate::RelayUrl
312pub trait Discovery: std::fmt::Debug + Send + Sync + 'static {
313    /// Publishes the given [`NodeData`] to the discovery mechanism.
314    ///
315    /// This is fire and forget, since the [`Endpoint`] can not wait for successful
316    /// publishing. If publishing is async, the implementation should start it's own task.
317    ///
318    /// This will be called from a tokio task, so it is safe to spawn new tasks.
319    /// These tasks will be run on the runtime of the [`super::Endpoint`].
320    fn publish(&self, _data: &NodeData) {}
321
322    /// Resolves the [`DiscoveryItem`] for the given [`NodeId`].
323    ///
324    /// Once the returned [`BoxStream`] is dropped, the service should stop any pending
325    /// work.
326    fn resolve(
327        &self,
328        _node_id: NodeId,
329    ) -> Option<BoxStream<Result<DiscoveryItem, DiscoveryError>>> {
330        None
331    }
332
333    /// Subscribe to all addresses that get *passively* discovered.
334    ///
335    /// An implementation may choose to defer emitting passively discovered nodes
336    /// until the stream is actually polled. To avoid missing discovered nodes,
337    /// poll the stream as soon as possible.
338    ///
339    /// If you do not regularly poll the stream, you may miss discovered nodes.
340    ///
341    /// Any discovery systems that only discover when explicitly resolving a
342    /// specific [`NodeId`] do not need to implement this method. Any nodes or
343    /// addresses that are discovered by calling `resolve` should NOT be added
344    /// to the `subscribe` stream.
345    ///
346    /// Discovery systems that are capable of receiving information about [`NodeId`]s
347    /// and their addressing information without explicitly calling `resolve`, i.e.,
348    /// systems that do "passive" discovery, should implement this method. If
349    /// `subscribe` is called multiple times, the passively discovered addresses
350    /// should be sent on all streams.
351    ///
352    /// The [`crate::endpoint::Endpoint`] will `subscribe` to the discovery system
353    /// and add the discovered addresses to the internal address book as they arrive
354    /// on this stream.
355    fn subscribe(&self) -> Option<BoxStream<DiscoveryEvent>> {
356        None
357    }
358}

372/// Node discovery results from [`Discovery`] services.
373///
374/// This is the item in the streams returned from [`Discovery::resolve`] and
375/// [`Discovery::subscribe`]. It contains the [`NodeData`] about the discovered node,
376/// and some additional metadata about the discovery.
377///
378/// This struct derefs to [`NodeData`], so you can access the methods from [`NodeData`]
379/// directly from [`DiscoveryItem`].
380#[derive(Debug, Clone, Eq, PartialEq)]
381pub struct DiscoveryItem {
382    /// The node info for the node, as discovered by the the discovery service.
383    node_info: NodeInfo,
384    /// A static string to identify the discovery source.
385    ///
386    /// Should be uniform per discovery service.
387    provenance: &'static str,
388    /// Optional timestamp when this node address info was last updated.
389    ///
390    /// Must be microseconds since the unix epoch.
391    // TODO(ramfox): this is currently unused. As we develop more `DiscoveryService`s, we may discover that we do not need this. It is only truly relevant when comparing `relay_urls`, since we can attempt to dial any number of socket addresses, but expect each node to have one "home relay" that we will attempt to contact them on. This means we would need some way to determine which relay url to choose between, if more than one relay url is reported.
392    last_updated: Option<u64>,
393}

459/// A discovery service that combines multiple discovery sources.
460///
461/// The discovery services will resolve concurrently.
462#[derive(Debug, Default)]
463pub struct ConcurrentDiscovery {
464    services: Vec<Box<dyn Discovery>>,
465}
466
467impl ConcurrentDiscovery {
468    /// Creates an empty [`ConcurrentDiscovery`].
469    pub fn empty() -> Self {
470        Self::default()
471    }
472
473    /// Creates a new [`ConcurrentDiscovery`].
474    pub fn from_services(services: Vec<Box<dyn Discovery>>) -> Self {
475        Self { services }
476    }
477
478    /// Adds a [`Discovery`] service.
479    pub fn add(&mut self, service: impl Discovery + 'static) {
480        self.services.push(Box::new(service));
481    }
482}

484impl<T> From<T> for ConcurrentDiscovery
485where
486    T: IntoIterator<Item = Box<dyn Discovery>>,
487{
488    fn from(iter: T) -> Self {
489        let services = iter.into_iter().collect::<Vec<_>>();
490        Self { services }
491    }
492}
493
494impl Discovery for ConcurrentDiscovery {
495    fn publish(&self, data: &NodeData) {
496        for service in &self.services {
497            service.publish(data);
498        }
499    }
500
501    fn resolve(&self, node_id: NodeId) -> Option<BoxStream<Result<DiscoveryItem, DiscoveryError>>> {
502        let streams = self
503            .services
504            .iter()
505            .filter_map(|service| service.resolve(node_id));
506
507        let streams = n0_future::MergeBounded::from_iter(streams);
508        Some(Box::pin(streams))
509    }
510
511    fn subscribe(&self) -> Option<BoxStream<DiscoveryEvent>> {
512        let mut streams = vec![];
513        for service in self.services.iter() {
514            if let Some(stream) = service.subscribe() {
515                streams.push(stream)
516            }
517        }
518
519        let streams = n0_future::MergeBounded::from_iter(streams);
520        Some(Box::pin(streams))
521    }
522}

524/// Maximum duration since the last control or data message received from an endpoint to make us
525/// start a discovery task.
526const MAX_AGE: Duration = Duration::from_secs(10);
527
528/// A wrapper around a tokio task which runs a node discovery.
529pub(super) struct DiscoveryTask {
530    on_first_rx: oneshot::Receiver<Result<(), DiscoveryError>>,
531    _task: AbortOnDropHandle<()>,
532}
533
534impl DiscoveryTask {
535    /// Starts a discovery task.
536    pub(super) fn start(ep: Endpoint, node_id: NodeId) -> Result<Self, DiscoveryError> {
537        ensure!(ep.discovery().is_some(), NoServiceConfiguredSnafu);
538        let (on_first_tx, on_first_rx) = oneshot::channel();
539        let me = ep.node_id();
540        let task = task::spawn(
541            async move { Self::run(ep, node_id, on_first_tx).await }.instrument(
542                error_span!("discovery", me = %me.fmt_short(), node = %node_id.fmt_short()),
543            ),
544        );
545        Ok(Self {
546            _task: AbortOnDropHandle::new(task),
547            on_first_rx,
548        })
549    }
550
551    /// Starts a discovery task after a delay and only if no path to the node was recently active.
552    ///
553    /// This returns `None` if we received data or control messages from the remote endpoint
554    /// recently enough. If not it returns a [`DiscoveryTask`].
555    ///
556    /// If `delay` is set, the [`DiscoveryTask`] will first wait for `delay` and then check again
557    /// if we recently received messages from remote endpoint. If true, the task will abort.
558    /// Otherwise, or if no `delay` is set, the discovery will be started.
559    pub(super) fn maybe_start_after_delay(
560        ep: &Endpoint,
561        node_id: NodeId,
562        delay: Option<Duration>,
563    ) -> Result<Option<Self>, DiscoveryError> {
564        // If discovery is not needed, don't even spawn a task.
565        if !Self::needs_discovery(ep, node_id) {
566            return Ok(None);
567        }
568        ensure!(ep.discovery().is_some(), NoServiceConfiguredSnafu);
569        let (on_first_tx, on_first_rx) = oneshot::channel();
570        let ep = ep.clone();
571        let me = ep.node_id();
572        let task = task::spawn(
573            async move {
574                // If delay is set, wait and recheck if discovery is needed. If not, early-exit.
575                if let Some(delay) = delay {
576                    time::sleep(delay).await;
577                    if !Self::needs_discovery(&ep, node_id) {
578                        debug!("no discovery needed, abort");
579                        on_first_tx.send(Ok(())).ok();
580                        return;
581                    }
582                }
583                Self::run(ep, node_id, on_first_tx).await
584            }
585            .instrument(
586                error_span!("discovery", me = %me.fmt_short(), node = %node_id.fmt_short()),
587            ),
588        );
589        Ok(Some(Self {
590            _task: AbortOnDropHandle::new(task),
591            on_first_rx,
592        }))
593    }
594
595    /// Waits until the discovery task produced at least one result.
596    pub(super) async fn first_arrived(&mut self) -> Result<(), DiscoveryError> {
597        let fut = &mut self.on_first_rx;
598        fut.await.expect("sender dropped")?;
599        Ok(())
600    }
601
602    fn create_stream(
603        ep: &Endpoint,
604        node_id: NodeId,
605    ) -> Result<BoxStream<Result<DiscoveryItem, DiscoveryError>>, DiscoveryError> {
606        let discovery = ep.discovery().ok_or(NoServiceConfiguredSnafu.build())?;
607        let stream = discovery
608            .resolve(node_id)
609            .ok_or(NoResultsSnafu { node_id }.build())?;
610        Ok(stream)
611    }
612
613    /// We need discovery if we have no paths to the node, or if the paths we do have
614    /// have timed out.
615    fn needs_discovery(ep: &Endpoint, node_id: NodeId) -> bool {
616        match ep.remote_info(node_id) {
617            // No info means no path to node -> start discovery.
618            None => true,
619            Some(info) => {
620                match (
621                    info.last_received(),
622                    info.relay_url.as_ref().and_then(|r| r.last_alive),
623                ) {
624                    // No path to node -> start discovery.
625                    (None, None) => true,
626                    // If we haven't received on direct addresses or the relay for MAX_AGE,
627                    // start discovery.
628                    (Some(elapsed), Some(elapsed_relay)) => {
629                        elapsed > MAX_AGE && elapsed_relay > MAX_AGE
630                    }
631                    (Some(elapsed), _) | (_, Some(elapsed)) => elapsed > MAX_AGE,
632                }
633            }
634        }
635    }
636
637    async fn run(
638        ep: Endpoint,
639        node_id: NodeId,
640        on_first_tx: oneshot::Sender<Result<(), DiscoveryError>>,
641    ) {
642        let mut stream = match Self::create_stream(&ep, node_id) {
643            Ok(stream) => stream,
644            Err(err) => {
645                on_first_tx.send(Err(err)).ok();
646                return;
647            }
648        };
649        let mut on_first_tx = Some(on_first_tx);
650        debug!("starting");
651        loop {
652            match stream.next().await {
653                Some(Ok(r)) => {
654                    let provenance = r.provenance;
655                    let node_addr = r.to_node_addr();
656                    if node_addr.is_empty() {
657                        debug!(%provenance, "empty address found");
658                        continue;
659                    }
660                    debug!(%provenance, addr = ?node_addr, "new address found");
661                    ep.add_node_addr_with_source(node_addr, provenance).ok();
662
663                    if let Some(tx) = on_first_tx.take() {
664                        tx.send(Ok(())).ok();
665                    }
666                    // Send the discovery item to the subscribers of the discovery broadcast stream.
667                    ep.discovery_subscribers()
668                        .send(DiscoveryEvent::Discovered(r));
669                }
670                Some(Err(err)) => {
671                    warn!(?err, "discovery service produced error");
672                    break;
673                }
674                None => break,
675            }
676        }
677        if let Some(tx) = on_first_tx.take() {
678            tx.send(Err(NoResultsSnafu { node_id }.build())).ok();
679        }
680    }
681}

683/// Error returned when a discovery watch stream lagged too far behind.
684///
685/// The stream returned from [`Endpoint::discovery_stream`] yields this error
686/// if the loop in which the stream is processed cannot keep up with the emitted
687/// discovery events. Attempting to read the next item from the channel afterwards
688/// will return the oldest [`DiscoveryItem`] that is still retained.
689///
690/// Includes the number of skipped messages.
691#[derive(Debug, Snafu)]
692#[snafu(display("channel lagged by {val}"))]
693pub struct Lagged {
694    /// The number of skipped messages
695    pub val: u64,
696}
697
698#[derive(Clone, Debug)]
699pub(super) struct DiscoverySubscribers {
700    inner: tokio::sync::broadcast::Sender<DiscoveryEvent>,
701}
702
703impl DiscoverySubscribers {
704    pub(crate) fn new() -> Self {
705        // TODO: Make capacity configurable from the endpoint builder?
706        // This is the maximum number of [`DiscoveryItem`]s held by the channel if
707        // subscribers are stalled.
708        const CAPACITY: usize = 128;
709        Self {
710            inner: tokio::sync::broadcast::Sender::new(CAPACITY),
711        }
712    }
713
714    pub(crate) fn subscribe(&self) -> impl Stream<Item = Result<DiscoveryEvent, Lagged>> + use<> {
715        use tokio_stream::wrappers::{BroadcastStream, errors::BroadcastStreamRecvError};
716        let recv = self.inner.subscribe();
717        BroadcastStream::new(recv).map_err(|BroadcastStreamRecvError::Lagged(n)| Lagged { val: n })
718    }
719
720    pub(crate) fn send(&self, item: DiscoveryEvent) {
721        // `broadcast::Sender::send` returns an error if the channel has no subscribers,
722        // which we don't care about.
723        self.inner.send(item).ok();
724    }
725}

1065    async fn new_endpoint(
1066        secret: SecretKey,
1067        disco: impl IntoDiscovery + 'static,
1068    ) -> (Endpoint, AbortOnDropHandle<Result<()>>) {
1069        let ep = Endpoint::builder()
1070            .secret_key(secret)
1071            .discovery(disco)
1072            .relay_mode(RelayMode::Disabled)
1073            .alpns(vec![TEST_ALPN.to_vec()])
1074            .bind()
1075            .await
1076            .unwrap();
1077
1078        let handle = tokio::spawn({
1079            let ep = ep.clone();
1080            async move {
1081                // Keep connections alive until the task is dropped.
1082                let mut connections = Vec::new();
1083                // we skip accept() errors, they can be caused by retransmits
1084                while let Some(connecting) = ep.accept().await.and_then(|inc| inc.accept().ok()) {
1085                    // Just accept incoming connections, but don't do anything with them.
1086                    let conn = connecting.await.context("connecting")?;
1087                    connections.push(conn);
1088                }
1089
1090                Ok::<_, Error>(())
1091            }
1092        });
1093
1094        (ep, AbortOnDropHandle::new(handle))
1095    }
