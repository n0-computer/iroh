1//! Node address discovery.
2//!
3//! To connect to an iroh node a [`NodeAddr`] is needed, which may contain a
4//! [`RelayUrl`] or one or more *direct addresses* in addition to the [`NodeId`].
5//!
6//! Since there is a conversion from [`NodeId`] to [`NodeAddr`], you can also use
7//! connect directly with a [`NodeId`].
8//!
9//! For this to work however, the endpoint has to get the addressing  information by
10//! other means.
11//!
12//! Node discovery is an automated system for an [`Endpoint`] to retrieve this addressing
13//! information.  Each iroh node will automatically publish their own addressing
14//! information.  Usually this means publishing which [`RelayUrl`] to use for their
15//! [`NodeId`], but they could also publish their direct addresses.
16//!
17//! The [`Discovery`] trait is used to define node discovery.  This allows multiple
18//! implementations to co-exist because there are many possible ways to implement this.
19//! Each [`Endpoint`] can use the discovery mechanisms most suitable to the application.
20//! The [`Builder::discovery`] method is used to add a discovery mechanism to an
21//! [`Endpoint`].
22//!
23//! Some generally useful discovery implementations are provided:
24//!
25//! - [`StaticProvider`] which allows application to add and remove out-of-band addressing
26//!   information.
27//!
28//! - The [`DnsDiscovery`] which performs lookups via the standard DNS systems.  To publish
29//!   to this DNS server a [`PkarrPublisher`] is needed.  [Number 0] runs a public instance
30//!   of a [`PkarrPublisher`] with attached DNS server which is globally available and a
31//!   reliable default choice.
32//!
33//! - The [`PkarrResolver`] which can perform lookups from designated [pkarr relay servers]
34//!   using HTTP.
35//!
36//! - [`MdnsDiscovery`]: mdns::MdnsDiscovery which uses the crate `swarm-discovery`, an
37//!   opinionated mDNS implementation, to discover nodes on the local network.
38//!
39//! - The [`DhtDiscovery`] also uses the [`pkarr`] system but can also publish and lookup
40//!   records to/from the Mainline DHT.
41//!
42//! To use multiple discovery systems simultaneously you can call [`Builder::discovery`].
43//! This will use [`ConcurrentDiscovery`] under the hood, which performs lookups to all
44//! discovery systems at the same time.
45//!
46//! [`Builder::discovery`] takes any type that implements [`IntoDiscovery`]. You can
47//! implement that trait on a builder struct if your discovery service needs information
48//! from the endpoint it is mounted on. After endpoint construction, your discovery service
49//! is built by calling [`IntoDiscovery::into_discovery`], passing the finished [`Endpoint`] to your
50//! builder.
51//!
52//! If your discovery service does not need any information from its endpoint, you can
53//! pass the discovery service directly to [`Builder::discovery`]: All types that
54//! implement [`Discovery`] also have a blanket implementation of [`IntoDiscovery`].
55//!
56//! # Examples
57//!
58//! A very common setup is to enable DNS discovery, which needs to be done in two parts as a
59//! [`PkarrPublisher`] and [`DnsDiscovery`]:
60//!
61//! ```no_run
62//! use iroh::{
63//!     Endpoint, SecretKey,
64//!     discovery::{dns::DnsDiscovery, pkarr::PkarrPublisher},
65//! };
66//!
67//! # async fn wrapper() -> n0_snafu::Result<()> {
68//! let ep = Endpoint::empty_builder()
69//!     .discovery(PkarrPublisher::n0_dns())
70//!     .discovery(DnsDiscovery::n0_dns())
71//!     .bind()
72//!     .await?;
73//! # Ok(())
74//! # }
75//! ```
76//!
77//! To also enable [`MdnsDiscovery`] it can be added as another service.
78//!
79//! ```no_run
80//! #[cfg(feature = "discovery-local-network")]
81//! # {
82//! # use iroh::{
83//! #    discovery::{dns::DnsDiscovery, pkarr::PkarrPublisher, mdns::MdnsDiscovery},
84//! #    Endpoint, SecretKey,
85//! # };
86//! #
87//! # async fn wrapper() -> n0_snafu::Result<()> {
88//! let ep = Endpoint::empty_builder()
89//!     .discovery(PkarrPublisher::n0_dns())
90//!     .discovery(DnsDiscovery::n0_dns())
91//!     .discovery(MdnsDiscovery::builder())
92//!     .bind()
93//!     .await?;
94//! # Ok(())
95//! # }
96//! # }
97//! ```
98//!
99//! [`NodeAddr`]: iroh_base::NodeAddr
100//! [`RelayUrl`]: crate::RelayUrl
101//! [`Builder::discovery`]: crate::endpoint::Builder::discovery
102//! [`DnsDiscovery`]: dns::DnsDiscovery
103//! [Number 0]: https://n0.computer
104//! [`PkarrResolver`]: pkarr::PkarrResolver
105//! [`PkarrPublisher`]: pkarr::PkarrPublisher
106//! [`DhtDiscovery`]: pkarr::dht::DhtDiscovery
107//! [pkarr relay servers]: https://pkarr.org/#servers
108//! [`MdnsDiscovery`]: mdns::MdnsDiscovery
109//! [`StaticProvider`]: static_provider::StaticProvider

136/// Trait for structs that can be converted into [`Discovery`].
137///
138/// This trait is implemented on builders for discovery services. Any type that implements this
139/// trait can be added as a discovery service in [`Builder::discovery`].
140///
141/// Any type that implements [`Discovery`] also implements [`IntoDiscovery`].
142///
143/// Iroh uses this trait to allow configuring the set of discovery services on the endpoint
144/// builder, while providing the discovery services access to information about the endpoint
145/// to [`IntoDiscovery::into_discovery`].
146///
147/// [`Builder::discovery`]: crate::endpoint::Builder::discovery
148pub trait IntoDiscovery: Send + Sync + std::fmt::Debug + 'static {
149    /// Turns this discovery builder into a ready-to-use discovery service.
150    ///
151    /// If an error is returned, building the endpoint will fail with this error.
152    fn into_discovery(self, endpoint: &Endpoint) -> Result<impl Discovery, IntoDiscoveryError>;
153}
154
155/// Blanket no-op impl of `IntoDiscovery` for `T: Discovery`.
156impl<T: Discovery> IntoDiscovery for T {
157    fn into_discovery(self, _endpoint: &Endpoint) -> Result<impl Discovery, IntoDiscoveryError> {
158        Ok(self)
159    }
160}

163pub(crate) trait DynIntoDiscovery: Send + Sync + std::fmt::Debug + 'static {
164    /// See [`IntoDiscovery::into_discovery`]
165    fn into_discovery(
166        self: Box<Self>,

168    ) -> Result<Box<dyn Discovery>, IntoDiscoveryError>;
169}

174        endpoint: &Endpoint,

178    }

256/// Node discovery for [`super::Endpoint`].
257///
258/// This trait defines publishing and resolving addressing information for a [`NodeId`].
259/// This enables connecting to other nodes with only knowing the [`NodeId`], by using this
260/// [`Discovery`] system to look up the actual addressing information.  It is common for
261/// implementations to require each node to publish their own information before it can be
262/// looked up by other nodes.
263///
264/// The published addressing information can include both a [`RelayUrl`] and/or direct
265/// addresses. See [`NodeData`] for details.
266///
267/// To allow for discovery, the [`super::Endpoint`] will call `publish` whenever
268/// discovery information changes. If a discovery mechanism requires a periodic
269/// refresh, it should start its own task.
270///
271/// [`RelayUrl`]: crate::RelayUrl
272pub trait Discovery: std::fmt::Debug + Send + Sync + 'static {
273    /// Publishes the given [`NodeData`] to the discovery mechanism.
274    ///
275    /// This is fire and forget, since the [`Endpoint`] can not wait for successful
276    /// publishing. If publishing is async, the implementation should start it's own task.
277    ///
278    /// This will be called from a tokio task, so it is safe to spawn new tasks.
279    /// These tasks will be run on the runtime of the [`super::Endpoint`].
280    fn publish(&self, _data: &NodeData) {}
281
282    /// Resolves the [`DiscoveryItem`] for the given [`NodeId`].
283    ///
284    /// Once the returned [`BoxStream`] is dropped, the service should stop any pending
285    /// work.
286    fn resolve(
287        &self,
288        _node_id: NodeId,
289    ) -> Option<BoxStream<Result<DiscoveryItem, DiscoveryError>>> {
290        None
291    }
292}
293
294impl<T: Discovery> Discovery for Arc<T> {
295    fn publish(&self, data: &NodeData) {
296        self.as_ref().publish(data);
297    }
298
299    fn resolve(&self, node_id: NodeId) -> Option<BoxStream<Result<DiscoveryItem, DiscoveryError>>> {
300        self.as_ref().resolve(node_id)
301    }
302}

304/// Node discovery results from [`Discovery`] services.
305///
306/// This is the item in the streams returned from [`Discovery::resolve`].
307/// It contains the [`NodeData`] about the discovered node,
308/// and some additional metadata about the discovery.
309///
310/// This struct derefs to [`NodeData`], so you can access the methods from [`NodeData`]
311/// directly from [`DiscoveryItem`].
312#[derive(Debug, Clone, Eq, PartialEq)]
313pub struct DiscoveryItem {
314    /// The node info for the node, as discovered by the the discovery service.
315    node_info: NodeInfo,
316    /// A static string to identify the discovery source.
317    ///
318    /// Should be uniform per discovery service.
319    provenance: &'static str,
320    /// Optional timestamp when this node address info was last updated.
321    ///
322    /// Must be microseconds since the unix epoch.
323    // TODO(ramfox): this is currently unused. As we develop more `DiscoveryService`s, we may discover that we do not need this. It is only truly relevant when comparing `relay_urls`, since we can attempt to dial any number of socket addresses, but expect each node to have one "home relay" that we will attempt to contact them on. This means we would need some way to determine which relay url to choose between, if more than one relay url is reported.
324    last_updated: Option<u64>,
325}

391/// A discovery service that combines multiple discovery sources.
392///
393/// The discovery services will resolve concurrently.
394#[derive(Debug, Default, Clone)]
395pub struct ConcurrentDiscovery {
396    services: Arc<RwLock<Vec<Box<dyn Discovery>>>>,
397}
398
399impl ConcurrentDiscovery {
400    /// Creates an empty [`ConcurrentDiscovery`].
401    pub fn empty() -> Self {
402        Self::default()
403    }
404
405    /// Creates a new [`ConcurrentDiscovery`].
406    pub fn from_services(services: Vec<Box<dyn Discovery>>) -> Self {
407        Self {
408            services: Arc::new(RwLock::new(services)),
409        }
410    }
411
412    /// Adds a [`Discovery`] service.
413    pub fn add(&self, service: impl Discovery + 'static) {
414        self.add_boxed(Box::new(service))
415    }
416
417    /// Adds an already `Box`ed [`Discovery`] service.
418    pub fn add_boxed(&self, service: Box<dyn Discovery>) {
419        self.services.write().expect("poisoned").push(service);
420    }
421
422    /// Is there any services configured?
423    pub fn is_empty(&self) -> bool {
424        self.services.read().expect("poisoned").is_empty()
425    }
426
427    /// How many services are configured?
428    pub fn len(&self) -> usize {
429        self.services.read().expect("poisoned").len()
430    }
431}
432
433impl<T> From<T> for ConcurrentDiscovery
434where
435    T: IntoIterator<Item = Box<dyn Discovery>>,
436{
437    fn from(iter: T) -> Self {
438        let services = iter.into_iter().collect::<Vec<_>>();
439        Self {
440            services: Arc::new(RwLock::new(services)),
441        }
442    }
443}
444
445impl Discovery for ConcurrentDiscovery {
446    fn publish(&self, data: &NodeData) {
447        let services = self.services.read().expect("poisoned");
448        for service in &*services {
449            service.publish(data);
450        }
451    }
452
453    fn resolve(&self, node_id: NodeId) -> Option<BoxStream<Result<DiscoveryItem, DiscoveryError>>> {
454        let services = self.services.read().expect("poisoned");
455        dbg!(services.len());
456        let streams = services
457            .iter()
458            .filter_map(|service| service.resolve(node_id));
459
460        let streams = n0_future::MergeBounded::from_iter(streams);
461        Some(Box::pin(streams))
462    }
463}

465/// Maximum duration since the last control or data message received from an endpoint to make us
466/// start a discovery task.
467const MAX_AGE: Duration = Duration::from_secs(10);
468
469/// A wrapper around a tokio task which runs a node discovery.
470pub(super) struct DiscoveryTask {
471    on_first_rx: oneshot::Receiver<Result<(), DiscoveryError>>,
472    _task: AbortOnDropHandle<()>,
473}
474
475impl DiscoveryTask {
476    /// Starts a discovery task.
477    pub(super) fn start(ep: Endpoint, node_id: NodeId) -> Result<Self, DiscoveryError> {
478        ensure!(!ep.discovery().is_empty(), NoServiceConfiguredSnafu);
479        let (on_first_tx, on_first_rx) = oneshot::channel();
480        let me = ep.node_id();
481        let task = task::spawn(
482            async move { Self::run(ep, node_id, on_first_tx).await }.instrument(
483                error_span!("discovery", me = %me.fmt_short(), node = %node_id.fmt_short()),
484            ),
485        );
486        Ok(Self {
487            _task: AbortOnDropHandle::new(task),
488            on_first_rx,
489        })
490    }
491
492    /// Starts a discovery task after a delay and only if no path to the node was recently active.
493    ///
494    /// This returns `None` if we received data or control messages from the remote endpoint
495    /// recently enough. If not it returns a [`DiscoveryTask`].
496    ///
497    /// If `delay` is set, the [`DiscoveryTask`] will first wait for `delay` and then check again
498    /// if we recently received messages from remote endpoint. If true, the task will abort.
499    /// Otherwise, or if no `delay` is set, the discovery will be started.
500    pub(super) fn maybe_start_after_delay(
501        ep: &Endpoint,
502        node_id: NodeId,
503        delay: Option<Duration>,
504    ) -> Result<Option<Self>, DiscoveryError> {
505        // If discovery is not needed, don't even spawn a task.
506        if !ep.needs_discovery(node_id, MAX_AGE) {
507            return Ok(None);
508        }
509        ensure!(!ep.discovery().is_empty(), NoServiceConfiguredSnafu);
510        let (on_first_tx, on_first_rx) = oneshot::channel();
511        let ep = ep.clone();
512        let me = ep.node_id();
513        let task = task::spawn(
514            async move {
515                // If delay is set, wait and recheck if discovery is needed. If not, early-exit.
516                if let Some(delay) = delay {
517                    time::sleep(delay).await;
518                    if !ep.needs_discovery(node_id, MAX_AGE) {
519                        debug!("no discovery needed, abort");
520                        on_first_tx.send(Ok(())).ok();
521                        return;
522                    }
523                }
524                Self::run(ep, node_id, on_first_tx).await
525            }
526            .instrument(
527                error_span!("discovery", me = %me.fmt_short(), node = %node_id.fmt_short()),
528            ),
529        );
530        Ok(Some(Self {
531            _task: AbortOnDropHandle::new(task),
532            on_first_rx,
533        }))
534    }
535
536    /// Waits until the discovery task produced at least one result.
537    pub(super) async fn first_arrived(&mut self) -> Result<(), DiscoveryError> {
538        let fut = &mut self.on_first_rx;
539        fut.await.expect("sender dropped")?;
540        Ok(())
541    }
542
543    fn create_stream(
544        ep: &Endpoint,
545        node_id: NodeId,
546    ) -> Result<BoxStream<Result<DiscoveryItem, DiscoveryError>>, DiscoveryError> {
547        ensure!(!ep.discovery().is_empty(), NoServiceConfiguredSnafu);
548        let stream = ep
549            .discovery()
550            .resolve(node_id)
551            .ok_or(NoResultsSnafu { node_id }.build())?;
552        Ok(stream)
553    }
554
555    async fn run(
556        ep: Endpoint,
557        node_id: NodeId,
558        on_first_tx: oneshot::Sender<Result<(), DiscoveryError>>,
559    ) {
560        let mut stream = match Self::create_stream(&ep, node_id) {
561            Ok(stream) => stream,
562            Err(err) => {
563                on_first_tx.send(Err(err)).ok();
564                return;
565            }
566        };
567        let mut on_first_tx = Some(on_first_tx);
568        debug!("starting");
569        loop {
570            match stream.next().await {
571                Some(Ok(r)) => {
572                    let provenance = r.provenance;
573                    let node_addr = r.to_node_addr();
574                    if node_addr.is_empty() {
575                        debug!(%provenance, "empty address found");
576                        continue;
577                    }
578                    debug!(%provenance, addr = ?node_addr, "new address found");
579                    ep.add_node_addr_with_source(node_addr, provenance).ok();
580
581                    if let Some(tx) = on_first_tx.take() {
582                        tx.send(Ok(())).ok();
583                    }
584                }
585                Some(Err(err)) => {
586                    warn!(?err, "discovery service produced error");
587                    break;
588                }
589                None => break,
590            }
591        }
592        if let Some(tx) = on_first_tx.take() {
593            tx.send(Err(NoResultsSnafu { node_id }.build())).ok();
594        }
595    }
596}

673                let ts = system_time_now() - 100_000;
674                let port: u16 = rand::rng().random_range(10_000..20_000);
675                // "240.0.0.0/4" is reserved and unreachable
676                let addr: SocketAddr = format!("240.0.0.1:{port}").parse().unwrap();
677                let data = NodeData::new(None, BTreeSet::from([addr]));
678                Some((data, ts))
679            } else {
680                self.shared.nodes.lock().unwrap().get(&node_id).cloned()
681            };
682            let stream = match addr_info {
683                Some((data, ts)) => {
684                    let item = DiscoveryItem::new(
685                        NodeInfo::from_parts(node_id, data),
686                        "test-disco",
687                        Some(ts),
688                    );
689                    let delay = self.delay;
690                    let fut = async move {
691                        time::sleep(delay).await;
692                        tracing::debug!("resolve: {} = {item:?}", node_id.fmt_short());
693                        Ok(item)
694                    };
695                    n0_future::stream::once_future(fut).boxed()
696                }
697                None => n0_future::stream::empty().boxed(),
698            };
699            Some(stream)
700        }
701    }

862    async fn new_endpoint<R: CryptoRng, D: Discovery + 'static, F: FnOnce(&Endpoint) -> D>(
863        rng: &mut R,
864        create_disco: F,
865    ) -> (Endpoint, AbortOnDropHandle<Result<()>>) {
866        new_endpoint_add(rng, |ep| {
867            let disco = create_disco(ep);
868            ep.discovery().add(disco);
869        })
870        .await
871    }

873    async fn new_endpoint_add<R: CryptoRng, F: FnOnce(&Endpoint)>(
874        rng: &mut R,
875        add_disco: F,
876    ) -> (Endpoint, AbortOnDropHandle<Result<()>>) {
877        let secret = SecretKey::generate(rng);
878
879        let ep = Endpoint::empty_builder()
880            .secret_key(secret)
881            .relay_mode(RelayMode::Disabled)
882            .alpns(vec![TEST_ALPN.to_vec()])
883            .bind()
884            .await
885            .unwrap();
886        add_disco(&ep);
887
888        let handle = tokio::spawn({
889            let ep = ep.clone();
890            async move {
891                // Keep connections alive until the task is dropped.
892                let mut connections = Vec::new();
893                // we skip accept() errors, they can be caused by retransmits
894                while let Some(connecting) = ep.accept().await.and_then(|inc| inc.accept().ok()) {
895                    // Just accept incoming connections, but don't do anything with them.
896                    let conn = connecting.await.context("connecting")?;
897                    connections.push(conn);
898                }
899
900                Ok::<_, Error>(())
901            }
902        });
903
904        (ep, AbortOnDropHandle::new(handle))
905    }
