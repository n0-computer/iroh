1//! Endpoint address discovery.
2//!
3//! To connect to an iroh endpoint a [`EndpointAddr`] is needed, which may contain a
4//! [`RelayUrl`] or one or more *direct addresses* in addition to the [`EndpointId`].
5//!
6//! Since there is a conversion from [`EndpointId`] to [`EndpointAddr`], you can also use
7//! connect directly with a [`EndpointId`].
8//!
9//! For this to work however, the endpoint has to get the addressing  information by
10//! other means.
11//!
12//! Endpoint discovery is an automated system for an [`Endpoint`] to retrieve this addressing
13//! information.  Each iroh endpoint will automatically publish their own addressing
14//! information.  Usually this means publishing which [`RelayUrl`] to use for their
15//! [`EndpointId`], but they could also publish their direct addresses.
16//!
17//! The [`Discovery`] trait is used to define endpoint discovery.  This allows multiple
18//! implementations to co-exist because there are many possible ways to implement this.
19//! Each [`Endpoint`] can use the discovery mechanisms most suitable to the application.
20//! The [`Builder::add_discovery`] method is used to add a discovery mechanism to an
21//! [`Endpoint`].
22//!
23//! Some generally useful discovery implementations are provided:
24//!
25//! - [`StaticProvider`] which allows application to add and remove out-of-band addressing
26//!   information.
27//!
28//! - The [`DnsDiscovery`] which performs lookups via the standard DNS systems.  To publish
29//!   to this DNS server a [`PkarrPublisher`] is needed.  [Number 0] runs a public instance
30//!   of a [`PkarrPublisher`] with attached DNS server which is globally available and a
31//!   reliable default choice.
32//!
33//! - The [`PkarrResolver`] which can perform lookups from designated [pkarr relay servers]
34//!   using HTTP.
35//!
36//! - [`MdnsDiscovery`]: mdns::MdnsDiscovery which uses the crate `swarm-discovery`, an
37//!   opinionated mDNS implementation, to discover endpoints on the local network.
38//!
39//! - The [`DhtDiscovery`] also uses the [`pkarr`] system but can also publish and lookup
40//!   records to/from the Mainline DHT.
41//!
42//! To use multiple discovery systems simultaneously you can call [`Builder::add_discovery`].
43//! This will use [`ConcurrentDiscovery`] under the hood, which performs lookups to all
44//! discovery systems at the same time.
45//!
46//! [`Builder::add_discovery`] takes any type that implements [`IntoDiscovery`]. You can
47//! implement that trait on a builder struct if your discovery service needs information
48//! from the endpoint it is mounted on. During endpoint construction, your discovery service
49//! is built by calling [`IntoDiscovery::into_discovery`], passing a [`DiscoveryContext`] to your
50//! builder. The [`DiscoveryContext`] gives access to the endpoint's secret key and DNS resolver.
51//!
52//! If your discovery service does not need any information from its endpoint, you can
53//! pass the discovery service directly to [`Builder::add_discovery`]: All types that
54//! implement [`Discovery`] also have a blanket implementation of [`IntoDiscovery`].
55//!
56//! # Examples
57//!
58//! A very common setup is to enable DNS discovery, which needs to be done in two parts as a
59//! [`PkarrPublisher`] and [`DnsDiscovery`]:
60//!
61//! ```no_run
62//! use iroh::{
63//!     Endpoint, SecretKey,
64//!     discovery::{dns::DnsDiscovery, pkarr::PkarrPublisher},
65//! };
66//!
67//! # async fn wrapper() -> n0_snafu::Result<()> {
68//! let ep = Endpoint::builder()
69//!     .add_discovery(PkarrPublisher::n0_dns())
70//!     .add_discovery(DnsDiscovery::n0_dns())
71//!     .bind()
72//!     .await?;
73//! # Ok(())
74//! # }
75//! ```
76//!
77//! To also enable [`MdnsDiscovery`] it can be added as another service.
78//!
79//! ```no_run
80//! #[cfg(feature = "discovery-local-network")]
81//! # {
82//! # use iroh::{
83//! #    discovery::{dns::DnsDiscovery, pkarr::PkarrPublisher, mdns::MdnsDiscovery},
84//! #    Endpoint, SecretKey,
85//! # };
86//! #
87//! # async fn wrapper() -> n0_snafu::Result<()> {
88//! let ep = Endpoint::builder()
89//!     .add_discovery(PkarrPublisher::n0_dns())
90//!     .add_discovery(DnsDiscovery::n0_dns())
91//!     .add_discovery(MdnsDiscovery::builder())
92//!     .bind()
93//!     .await?;
94//! # Ok(())
95//! # }
96//! # }
97//! ```
98//!
99//! [`EndpointAddr`]: iroh_base::EndpointAddr
100//! [`RelayUrl`]: crate::RelayUrl
101//! [`Builder::add_discovery`]: crate::endpoint::Builder::add_discovery
102//! [`DnsDiscovery`]: dns::DnsDiscovery
103//! [Number 0]: https://n0.computer
104//! [`PkarrResolver`]: pkarr::PkarrResolver
105//! [`PkarrPublisher`]: pkarr::PkarrPublisher
106//! [`DhtDiscovery`]: pkarr::dht::DhtDiscovery
107//! [pkarr relay servers]: https://pkarr.org/#servers
108//! [`MdnsDiscovery`]: mdns::MdnsDiscovery
109//! [`StaticProvider`]: static_provider::StaticProvider

138/// Trait for structs that can be converted into [`Discovery`].
139///
140/// This trait is implemented on builders for discovery services. Any type that implements this
141/// trait can be added as a discovery service in [`Builder::add_discovery`].
142///
143/// Any type that implements [`Discovery`] also implements [`IntoDiscovery`].
144///
145/// Iroh uses this trait to allow configuring the set of discovery services on the endpoint
146/// builder, while providing the discovery services access to information about the endpoint
147/// creation via the [`DiscoveryContext`] parameter to [`IntoDiscovery::into_discovery`].
148///
149/// [`Builder::add_discovery`]: crate::endpoint::Builder::add_discovery
150pub trait IntoDiscovery: Send + Sync + std::fmt::Debug + 'static {
151    /// Turns this discovery builder into a ready-to-use discovery service.
152    ///
153    /// The [`DiscoveryContext`] contains information about the [`Endpoint`] onto which this
154    /// discovery service is being added. It can be used by discovery services that need
155    /// a DNS resolver, or the endpoint's secret key to sign messages.
156    ///
157    /// If an error is returned, building the endpoint will fail with this error.
158    fn into_discovery(
159        self,
160        context: &DiscoveryContext,
161    ) -> Result<impl Discovery, IntoDiscoveryError>;
162}
163
164/// Blanket no-op impl of `IntoDiscovery` for `T: Discovery`.
165impl<T: Discovery> IntoDiscovery for T {
166    fn into_discovery(
167        self,
168        _context: &DiscoveryContext,
169    ) -> Result<impl Discovery, IntoDiscoveryError> {
170        Ok(self)
171    }
172}

294/// Endpoint discovery for [`super::Endpoint`].
295///
296/// This trait defines publishing and resolving addressing information for a [`EndpointId`].
297/// This enables connecting to other endpoints with only knowing the [`EndpointId`], by using this
298/// [`Discovery`] system to look up the actual addressing information.  It is common for
299/// implementations to require each endpoint to publish their own information before it can be
300/// looked up by other endpoints.
301///
302/// The published addressing information can include both a [`RelayUrl`] and/or direct
303/// addresses. See [`EndpointData`] for details.
304///
305/// To allow for discovery, the [`super::Endpoint`] will call `publish` whenever
306/// discovery information changes. If a discovery mechanism requires a periodic
307/// refresh, it should start its own task.
308///
309/// [`RelayUrl`]: crate::RelayUrl
310pub trait Discovery: std::fmt::Debug + Send + Sync + 'static {
311    /// Publishes the given [`EndpointData`] to the discovery mechanism.
312    ///
313    /// This is fire and forget, since the [`Endpoint`] can not wait for successful
314    /// publishing. If publishing is async, the implementation should start it's own task.
315    ///
316    /// This will be called from a tokio task, so it is safe to spawn new tasks.
317    /// These tasks will be run on the runtime of the [`super::Endpoint`].
318    fn publish(&self, _data: &EndpointData) {}
319
320    /// Resolves the [`DiscoveryItem`] for the given [`EndpointId`].
321    ///
322    /// Once the returned [`BoxStream`] is dropped, the service should stop any pending
323    /// work.
324    fn resolve(
325        &self,
326        _endpoint_id: EndpointId,
327    ) -> Option<BoxStream<Result<DiscoveryItem, DiscoveryError>>> {
328        None
329    }
330}
331
332impl<T: Discovery> Discovery for Arc<T> {
333    fn publish(&self, data: &EndpointData) {
334        self.as_ref().publish(data);
335    }
336
337    fn resolve(
338        &self,
339        endpoint_id: EndpointId,
340    ) -> Option<BoxStream<Result<DiscoveryItem, DiscoveryError>>> {
341        self.as_ref().resolve(endpoint_id)
342    }
343}

345/// Endpoint discovery results from [`Discovery`] services.
346///
347/// This is the item in the streams returned from [`Discovery::resolve`].
348/// It contains the [`EndpointData`] about the discovered endpoint,
349/// and some additional metadata about the discovery.
350///
351/// This struct derefs to [`EndpointData`], so you can access the methods from [`EndpointData`]
352/// directly from [`DiscoveryItem`].
353#[derive(Debug, Clone, Eq, PartialEq)]
354pub struct DiscoveryItem {
355    /// The endpoint info for the endpoint, as discovered by the the discovery service.
356    endpoint_info: EndpointInfo,
357    /// A static string to identify the discovery source.
358    ///
359    /// Should be uniform per discovery service.
360    provenance: &'static str,
361    /// Optional timestamp when this endpoint address info was last updated.
362    ///
363    /// Must be microseconds since the unix epoch.
364    // TODO(ramfox): this is currently unused. As we develop more `DiscoveryService`s, we may discover that we do not need this. It is only truly relevant when comparing `relay_urls`, since we can attempt to dial any number of socket addresses, but expect each endpoint to have one "home relay" that we will attempt to contact them on. This means we would need some way to determine which relay url to choose between, if more than one relay url is reported.
365    last_updated: Option<u64>,
366}

436/// A discovery service that combines multiple discovery sources.
437///
438/// The discovery services will resolve concurrently.
439#[derive(Debug, Default, Clone)]
440pub struct ConcurrentDiscovery {
441    services: Arc<RwLock<Vec<Box<dyn Discovery>>>>,
442    /// The data last published, used to publish when adding a new service.
443    last_data: Arc<RwLock<Option<EndpointData>>>,
444}
445
446impl ConcurrentDiscovery {
447    /// Creates an empty [`ConcurrentDiscovery`].
448    pub fn empty() -> Self {
449        Self::default()
450    }
451
452    /// Creates a new [`ConcurrentDiscovery`].
453    pub fn from_services(services: Vec<Box<dyn Discovery>>) -> Self {
454        Self {
455            services: Arc::new(RwLock::new(services)),
456            last_data: Default::default(),
457        }
458    }
459
460    /// Adds a [`Discovery`] service.
461    ///
462    /// If there is historical discovery data, it will be published immediately on this service.
463    pub fn add(&self, service: impl Discovery + 'static) {
464        self.add_boxed(Box::new(service))
465    }
466
467    /// Adds an already `Box`ed [`Discovery`] service.
468    ///
469    /// If there is historical discovery data, it will be published immediately on this service.
470    pub fn add_boxed(&self, service: Box<dyn Discovery>) {
471        {
472            let data = self.last_data.read().expect("poisoned");
473            if let Some(data) = &*data {
474                service.publish(data)
475            }
476        }
477        self.services.write().expect("poisoned").push(service);
478    }
479
480    /// Is there any services configured?
481    pub fn is_empty(&self) -> bool {
482        self.services.read().expect("poisoned").is_empty()
483    }
484
485    /// How many services are configured
486    pub fn len(&self) -> usize {
487        self.services.read().expect("poisoned").len()
488    }
489}
490
491impl<T> From<T> for ConcurrentDiscovery
492where
493    T: IntoIterator<Item = Box<dyn Discovery>>,
494{
495    fn from(iter: T) -> Self {
496        let services = iter.into_iter().collect::<Vec<_>>();
497        Self {
498            services: Arc::new(RwLock::new(services)),
499            last_data: Default::default(),
500        }
501    }
502}
503
504impl Discovery for ConcurrentDiscovery {
505    fn publish(&self, data: &EndpointData) {
506        let services = self.services.read().expect("poisoned");
507        for service in &*services {
508            service.publish(data);
509        }
510
511        self.last_data
512            .write()
513            .expect("poisoned")
514            .replace(data.clone());
515    }
516
517    fn resolve(
518        &self,
519        endpoint_id: EndpointId,
520    ) -> Option<BoxStream<Result<DiscoveryItem, DiscoveryError>>> {
521        let services = self.services.read().expect("poisoned");
522        let streams = services
523            .iter()
524            .filter_map(|service| service.resolve(endpoint_id));
525
526        let streams = n0_future::MergeBounded::from_iter(streams);
527        Some(Box::pin(streams))
528    }
529}

531/// Maximum duration since the last control or data message received from an endpoint to make us
532/// start a discovery task.
533const MAX_AGE: Duration = Duration::from_secs(10);
534
535/// A wrapper around a tokio task which runs an endpoint discovery.
536pub(super) struct DiscoveryTask {
537    on_first_rx: oneshot::Receiver<Result<(), DiscoveryError>>,
538    _task: AbortOnDropHandle<()>,
539}
540
541impl DiscoveryTask {
542    /// Starts a discovery task.
543    pub(super) fn start(ep: Endpoint, endpoint_id: EndpointId) -> Result<Self, DiscoveryError> {
544        ensure!(!ep.discovery().is_empty(), NoServiceConfiguredSnafu);
545        let (on_first_tx, on_first_rx) = oneshot::channel();
546        let me = ep.id();
547        let task = task::spawn(
548            async move { Self::run(ep, endpoint_id, on_first_tx).await }.instrument(
549                error_span!("discovery", me = %me.fmt_short(), endpoint = %endpoint_id.fmt_short()),
550            ),
551        );
552        Ok(Self {
553            _task: AbortOnDropHandle::new(task),
554            on_first_rx,
555        })
556    }
557
558    /// Starts a discovery task after a delay and only if no path to the endpoint was recently active.
559    ///
560    /// This returns `None` if we received data or control messages from the remote endpoint
561    /// recently enough. If not it returns a [`DiscoveryTask`].
562    ///
563    /// If `delay` is set, the [`DiscoveryTask`] will first wait for `delay` and then check again
564    /// if we recently received messages from remote endpoint. If true, the task will abort.
565    /// Otherwise, or if no `delay` is set, the discovery will be started.
566    pub(super) fn maybe_start_after_delay(
567        ep: &Endpoint,
568        endpoint_id: EndpointId,
569        delay: Option<Duration>,
570    ) -> Result<Option<Self>, DiscoveryError> {
571        // If discovery is not needed, don't even spawn a task.
572        if !ep.needs_discovery(endpoint_id, MAX_AGE) {
573            return Ok(None);
574        }
575        ensure!(!ep.discovery().is_empty(), NoServiceConfiguredSnafu);
576        let (on_first_tx, on_first_rx) = oneshot::channel();
577        let ep = ep.clone();
578        let me = ep.id();
579        let task = task::spawn(
580            async move {
581                // If delay is set, wait and recheck if discovery is needed. If not, early-exit.
582                if let Some(delay) = delay {
583                    time::sleep(delay).await;
584                    if !ep.needs_discovery(endpoint_id, MAX_AGE) {
585                        debug!("no discovery needed, abort");
586                        on_first_tx.send(Ok(())).ok();
587                        return;
588                    }
589                }
590                Self::run(ep, endpoint_id, on_first_tx).await
591            }
592            .instrument(
593                error_span!("discovery", me = %me.fmt_short(), endpoint = %endpoint_id.fmt_short()),
594            ),
595        );
596        Ok(Some(Self {
597            _task: AbortOnDropHandle::new(task),
598            on_first_rx,
599        }))
600    }
601
602    /// Waits until the discovery task produced at least one result.
603    pub(super) async fn first_arrived(&mut self) -> Result<(), DiscoveryError> {
604        let fut = &mut self.on_first_rx;
605        fut.await.expect("sender dropped")?;
606        Ok(())
607    }
608
609    fn create_stream(
610        ep: &Endpoint,
611        endpoint_id: EndpointId,
612    ) -> Result<BoxStream<Result<DiscoveryItem, DiscoveryError>>, DiscoveryError> {
613        ensure!(!ep.discovery().is_empty(), NoServiceConfiguredSnafu);
614        let stream = ep
615            .discovery()
616            .resolve(endpoint_id)
617            .ok_or(NoResultsSnafu { endpoint_id }.build())?;
618        Ok(stream)
619    }
620
621    async fn run(
622        ep: Endpoint,
623        endpoint_id: EndpointId,
624        on_first_tx: oneshot::Sender<Result<(), DiscoveryError>>,
625    ) {
626        let mut stream = match Self::create_stream(&ep, endpoint_id) {
627            Ok(stream) => stream,
628            Err(err) => {
629                on_first_tx.send(Err(err)).ok();
630                return;
631            }
632        };
633        let mut on_first_tx = Some(on_first_tx);
634        debug!("starting");
635        loop {
636            match stream.next().await {
637                Some(Ok(r)) => {
638                    let provenance = r.provenance;
639                    let endpoint_addr = r.to_endpoint_addr();
640                    if endpoint_addr.is_empty() {
641                        debug!(%provenance, "empty address found");
642                        continue;
643                    }
644                    debug!(%provenance, addr = ?endpoint_addr, "new address found");
645                    let source = crate::magicsock::Source::Discovery {
646                        name: provenance.to_string(),
647                    };
648                    ep.add_endpoint_addr(endpoint_addr, source).ok();
649
650                    if let Some(tx) = on_first_tx.take() {
651                        tx.send(Ok(())).ok();
652                    }
653                }
654                Some(Err(err)) => {
655                    warn!(?err, "discovery service produced error");
656                    break;
657                }
658                None => break,
659            }
660        }
661        if let Some(tx) = on_first_tx.take() {
662            tx.send(Err(NoResultsSnafu { endpoint_id }.build())).ok();
663        }

726            if !self.publish {
727                return;
728            }

731                .endpoints
732                .lock()
733                .unwrap()

738            &self,
739            endpoint_id: EndpointId,
740        ) -> Option<BoxStream<Result<DiscoveryItem, DiscoveryError>>> {
741            let addr_info = if self.resolve_wrong {
742                let ts = system_time_now() - 100_000;
743                let port: u16 = rand::rng().random_range(10_000..20_000);
744                // "240.0.0.0/4" is reserved and unreachable
745                let addr: SocketAddr = format!("240.0.0.1:{port}").parse().unwrap();
746                let data = EndpointData::new(None, BTreeSet::from([addr]));
747                Some((data, ts))
748            } else {
749                self.shared
750                    .endpoints
751                    .lock()
752                    .unwrap()
753                    .get(&endpoint_id)
754                    .cloned()
755            };
756            let stream = match addr_info {
757                Some((data, ts)) => {
758                    let item = DiscoveryItem::new(
759                        EndpointInfo::from_parts(endpoint_id, data),
760                        "test-disco",
761                        Some(ts),
762                    );
763                    let delay = self.delay;
764                    let fut = async move {
765                        time::sleep(delay).await;
766                        tracing::debug!("resolve: {} = {item:?}", endpoint_id.fmt_short());
767                        Ok(item)
768                    };
769                    n0_future::stream::once_future(fut).boxed()
770                }
771                None => n0_future::stream::empty().boxed(),
772            };
773            Some(stream)
774        }

958    async fn new_endpoint(
959        secret: SecretKey,
960        disco: impl IntoDiscovery + 'static,
961    ) -> (Endpoint, AbortOnDropHandle<Result<()>>) {
962        let ep = Endpoint::builder()
963            .secret_key(secret)
964            .add_discovery(disco)
965            .relay_mode(RelayMode::Disabled)
966            .alpns(vec![TEST_ALPN.to_vec()])
967            .bind()
968            .await
969            .unwrap();
970
971        let handle = tokio::spawn({
972            let ep = ep.clone();
973            async move {
974                // Keep connections alive until the task is dropped.
975                let mut connections = Vec::new();
976                // we skip accept() errors, they can be caused by retransmits
977                while let Some(connecting) = ep.accept().await.and_then(|inc| inc.accept().ok()) {
978                    // Just accept incoming connections, but don't do anything with them.
979                    let conn = connecting.await.context("connecting")?;
980                    connections.push(conn);
981                }
982
983                Ok::<_, Error>(())
984            }
985        });
986
987        (ep, AbortOnDropHandle::new(handle))
988    }
