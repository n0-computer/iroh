1//! The actor that runs network probes and publishes a [`Report`].
2//!
3//! [`NetReportActor`] runs in the background for as long as the endpoint is
4//! alive. It learns about the network in three ways and writes what it finds
5//! into a [`Report`] that callers watch:
6//!
7//! - QAD (QUIC Address Discovery) opens a QUIC connection to a relay. The
8//!   relay reports the public address it sees us coming from, and the round
9//!   trip measures our latency to that relay.
10//! - An HTTPS probe measures latency to a relay with a plain GET request. It
11//!   finds no address, but it still works on networks that block QUIC.
12//! - The captive portal check looks for a network that intercepts HTTP.
13//!
14//! QAD connections are long-lived, and that shapes the rest. For each
15//! address family the actor races a probe to several relays, keeps the
16//! connection that answers first, and closes the others. It then holds that
17//! connection open. The relay only sees our address from the packets we
18//! send, so a change is not reported the instant it happens: the
19//! connection's keep-alive prompts the relay to re-observe within seconds,
20//! and a network change we notice ourselves starts a fresh probe at once.
21//! Either way the update arrives on the same [`ProbeEvent`] channel as probe
22//! results, so everything the actor learns lands in one place.
23//!
24//! A probe cycle is a round of probing triggered by a request. Because the
25//! open QAD connections already keep addresses current, a cycle is mostly
26//! about the rest: measuring latency to every relay over HTTPS, picking the
27//! preferred relay, and checking for a captive portal. [`ProbeScope`] sets
28//! how much of that a cycle does, and [`ProbeCycle`] holds the one in
29//! flight. A cycle publishes a first report within [`FIRST_REPORT_TIMEOUT`]
30//! even while probes are still running, and gives up on any stragglers after
31//! [`ABORT_TIMEOUT`].

94/// A probe request waiting for the actor to pick it up.
95///
96/// Several [`Client::run_probes`](super::Client::run_probes) calls can
97/// arrive before the actor handles them. They collapse into this one
98/// request, which takes the most urgent [`ProbeScope`] of the batch and
99/// waits here until the actor takes it.
100pub(super) struct ProbeRequest {
101    pub if_state: IfState,
102    pub scope: ProbeScope,
103}
104
105/// A one-slot mailbox that carries a [`ProbeRequest`] from a
106/// [`Client`](super::Client) to the [`NetReportActor`].
107///
108/// It holds at most one request. A second request that arrives before the
109/// actor has taken the first does not queue behind it; it merges into it,
110/// raising the [`ProbeScope`] to the more urgent of the two and keeping the
111/// newer interface state. This is why it is a hand-written slot and not a
112/// channel: a channel would queue the requests, and probing twice in a row
113/// wastes work when one probe with the combined scope would do.
114pub(super) struct RequestSlot {
115    slot: std::sync::Mutex<Option<ProbeRequest>>,
116    notify: tokio::sync::Notify,
117}
118
119impl std::fmt::Debug for RequestSlot {
120    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
121        f.debug_struct("RequestSlot").finish_non_exhaustive()
122    }
123}

125impl RequestSlot {
126    pub(super) fn new() -> Self {
127        Self {
128            slot: std::sync::Mutex::new(None),
129            notify: tokio::sync::Notify::new(),
130        }
131    }
132
133    /// Merges a probe request into the slot.
134    ///
135    /// If a request is already pending, the scope is escalated to the more
136    /// urgent of the two and `if_state` is overwritten with the latest
137    /// value. If no request is pending, a new one is created.
138    pub(super) fn request(&self, if_state: IfState, scope: ProbeScope) {
139        let mut guard = self.slot.lock().expect("not poisoned");
140        match guard.as_mut() {
141            Some(pending) => {
142                pending.scope = pending.scope.max(scope);
143                pending.if_state = if_state;
144            }
145            None => {
146                *guard = Some(ProbeRequest { if_state, scope });
147            }
148        }
149        drop(guard);
150        self.notify.notify_one();
151    }
152
153    /// Takes the pending request, leaving the slot empty.
154    fn take(&self) -> Option<ProbeRequest> {
155        self.slot.lock().expect("not poisoned").take()
156    }
157}

159/// A message from a probe task to the [`NetReportActor`].
160///
161/// Each probe a cycle starts sends exactly one final result: `QadResult`,
162/// `Https`, or `CaptivePortal`. The actor counts these to know when the
163/// cycle is done. An open QAD connection is different: it sends a
164/// `QadObservation` every time our address changes, for as long as it stays
165/// open, and those belong to no cycle.
166pub(super) enum ProbeEvent {
167    /// A QAD probe finished. On success it carries the connection to keep
168    /// open.
169    #[cfg(not(wasm_browser))]
170    QadResult(
171        AddrFamily,
172        ProbeOutcome<(QadProbeReport, QadConn), QadProbeError>,
173    ),
174    /// An open QAD connection reported an address, possibly a new one.
175    #[cfg(not(wasm_browser))]
176    QadObservation(AddrFamily, QadProbeReport),
177    /// An HTTPS latency probe finished.
178    Https(ProbeOutcome<HttpsProbeReport, MeasureHttpsLatencyError>),
179    /// The captive portal check finished.
180    #[cfg(not(wasm_browser))]
181    CaptivePortal(ProbeOutcome<bool, CaptivePortalError>),
182}

184/// How much of the probe set a cycle runs.
185///
186/// The scope plays two roles. On the request that starts a cycle it says how
187/// urgent the trigger is: a `Full` request comes from a real network change,
188/// so it aborts any cycle in progress and starts over, while a `Refresh`
189/// request waits for the current cycle to finish. On the cycle itself it
190/// says how much to probe.
191///
192/// The open QAD connections keep our address up to date on their own, within
193/// their keep-alive interval (see the [module docs](self)), so neither scope
194/// has to probe QAD just to stay current. The difference is what else a cycle
195/// does:
196///
197/// - `Full` throws away the open QAD connections and starts from nothing. It
198///   opens a fresh QAD connection for every available family, measures
199///   latency to every relay over HTTPS, and runs the captive portal check.
200///   This is what a real network change calls for.
201/// - `Refresh` keeps the open QAD connections and only does the work they do
202///   not cover. It re-measures relay latency over HTTPS and re-picks the
203///   preferred relay, and it opens a QAD connection only for a family that
204///   has none, because its connection dropped or its interface just came up.
205///   It skips the captive portal check.
206///
207/// A `Refresh` request can still turn into a `Full` cycle: the actor forces
208/// one when the full-report interval has elapsed, or when the last report
209/// found a captive portal and no working UDP.
210///
211/// The variants order `Refresh < Full` so that merging two pending requests
212/// can just take the more urgent one with `max`.
213#[derive(Debug, Clone, Copy, PartialEq, Eq, PartialOrd, Ord)]
214pub(crate) enum ProbeScope {
215    /// Keep the open QAD connections. Re-measure relay latency and reconnect
216    /// any family that has no open connection.
217    Refresh,
218    /// Close the open QAD connections and run every probe from scratch.
219    Full,
220}
221
222impl ProbeScope {
223    /// Maps a netmon "is major change" flag to a [`ProbeScope`].
224    pub(crate) fn from_major(is_major: bool) -> Self {
225        if is_major { Self::Full } else { Self::Refresh }
226    }
227}

229/// Per-family QAD progress within a cycle.
230#[derive(Debug, Clone, Copy, Default)]
231struct QadFamily {
232    /// This cycle is probing the family.
233    probing: bool,
234    /// The family is determined: it has an address, or all of its probes have
235    /// finished (so the family is genuinely down).

238    #[cfg(not(wasm_browser))]

244    fn new(probes: usize) -> Self {

254/// QAD progress for both address families within a cycle.
255#[derive(Debug, Clone, Copy, Default)]
256struct QadState {
257    v4: QadFamily,
258    v6: QadFamily,
259}
260
261impl QadState {
262    /// Builds the cycle's QAD gate state from the number of probes started per
263    /// family.
264    fn new(v4_probes: usize, v6_probes: usize) -> Self {
265        QadState {
266            v4: QadFamily::new(v4_probes),
267            v6: QadFamily::new(v6_probes),
268        }
269    }
270
271    /// Returns `true` once every family this cycle is probing is determined.
272    /// The first report is held until this holds (or [`FIRST_REPORT_TIMEOUT`]
273    /// fires), so consumers do not see a v4-only report a moment before v6.
274    fn all_determined(&self) -> bool {
275        (!self.v4.probing || self.v4.determined) && (!self.v6.probing || self.v6.determined)
276    }
277
278    /// Records that one of `family`'s probes finished with `outcome`.
279    ///
280    /// The family becomes determined on its first success or once all of its
281    /// probes are done, so a fast failure does not gate a premature
282    /// family-negative first report.
283    #[cfg(not(wasm_browser))]
284    fn record_result<T, E>(&mut self, family: AddrFamily, outcome: &ProbeOutcome<T, E>) {
285        let fam = self.family_mut(family);
286        fam.outstanding = fam.outstanding.saturating_sub(1);
287        if matches!(outcome, ProbeOutcome::Ok(_)) || fam.outstanding == 0 {
288            fam.determined = true;
289        }
290    }
291
292    #[cfg(not(wasm_browser))]
293    fn family_mut(&mut self, family: AddrFamily) -> &mut QadFamily {
294        match family {
295            AddrFamily::V4 => &mut self.v4,
296            AddrFamily::V6 => &mut self.v6,
297        }
298    }
299}

301/// Tracks recent reports for preferred-relay selection and full-cycle
302/// cadence.
303#[derive(Debug)]
304struct ReportHistory {
305    /// When true, the next cycle is forced to be `Full` rather than `Refresh`.
306    next_full: bool,
307    /// Reports from the last five minutes, keyed by completion time.
308    prev: std::collections::BTreeMap<Instant, Report>,
309    /// The most recent completed report.
310    last: Option<Report>,
311    /// Time of the last `Full` report.
312    last_full: Instant,
313}
314
315impl Default for ReportHistory {
316    fn default() -> Self {
317        Self {
318            next_full: true,
319            prev: Default::default(),
320            last: Default::default(),
321            last_full: Instant::now(),
322        }
323    }
324}
325
326impl ReportHistory {
327    /// Records `r` and sets `r.preferred_relay` to the best candidate
328    /// across the last five minutes of reports.
329    ///
330    /// Applies hysteresis: the preferred relay only changes when the new
331    /// candidate is at least 33% faster than the current one.
332    fn record(&mut self, r: &mut Report) {
333        let mut prev_relay = None;
334        if let Some(ref last) = self.last {
335            prev_relay.clone_from(&last.preferred_relay);
336
337            if r.mapping_varies_by_dest_ipv4.is_none() {
338                r.mapping_varies_by_dest_ipv4 = last.mapping_varies_by_dest_ipv4;
339            }
340            if r.mapping_varies_by_dest_ipv6.is_none() {
341                r.mapping_varies_by_dest_ipv6 = last.mapping_varies_by_dest_ipv6;
342            }
343        }
344
345        let now = Instant::now();
346        const MAX_AGE: Duration = Duration::from_secs(5 * 60);
347
348        let mut best_recent = RelayLatencies::default();
349
350        let mut to_remove = Vec::new();
351        for (t, pr) in self.prev.iter() {
352            if now.duration_since(*t) > MAX_AGE {
353                to_remove.push(*t);
354                continue;
355            }
356            best_recent.merge(&pr.relay_latency);
357        }
358        best_recent.merge(&r.relay_latency);
359
360        for t in to_remove {
361            self.prev.remove(&t);
362        }
363
364        let mut best_any = Duration::default();
365        let mut old_relay_cur_latency = Duration::default();
366        for (_, url, duration) in r.relay_latency.iter() {
367            if Some(url) == prev_relay.as_ref() {
368                old_relay_cur_latency = duration;
369            }
370            if let Some(best) = best_recent.get(url)
371                && (r.preferred_relay.is_none() || best < best_any)
372            {
373                best_any = best;
374                r.preferred_relay.replace(url.clone());
375            }
376        }
377
378        // Hysteresis: don't switch if the new relay isn't much better.
379        if prev_relay.is_some()
380            && r.preferred_relay != prev_relay
381            && !old_relay_cur_latency.is_zero()
382            && best_any > old_relay_cur_latency / 3 * 2
383        {
384            r.preferred_relay = prev_relay;
385        }
386
387        self.prev.insert(now, r.clone());
388        self.last = Some(r.clone());
389    }
390
391    /// Decides the scope of the next cycle.
392    ///
393    /// A `Full` request always yields a `Full` cycle. A `Refresh` request is
394    /// promoted to `Full` on the first cycle, once the full-report interval has
395    /// elapsed, or when the last report saw a captive portal without UDP.
396    fn scope_for(&self, request: ProbeScope, now: Instant) -> ProbeScope {
397        let full = request == ProbeScope::Full
398            || self.next_full
399            || now.duration_since(self.last_full) > FULL_REPORT_INTERVAL
400            || self
401                .last
402                .as_ref()
403                .is_some_and(|r| !r.has_udp() && r.captive_portal == Some(true));
404        ProbeScope::from_major(full)
405    }
406}

525    /// Runs the actor until the shutdown token is cancelled.
526    ///
527    /// On shutdown, dropping `self` drops the [`JoinSet`] and the QAD
528    /// connections, aborting every task the actor owns.
529    pub(super) async fn run(mut self) {
530        loop {
531            let report_deadline = match self.cycle.as_ref().and_then(|c| c.report_deadline) {
532                Some(t) => MaybeFuture::Some(time::sleep_until(t)),
533                None => MaybeFuture::None,
534            };
535            let abort_deadline = match self.cycle.as_ref().and_then(|c| c.abort_deadline) {
536                Some(t) => MaybeFuture::Some(time::sleep_until(t)),
537                None => MaybeFuture::None,
538            };
539            n0_future::pin!(report_deadline);
540            n0_future::pin!(abort_deadline);
541
542            tokio::select! {
543                biased;
544
545                _ = self.shutdown.cancelled() => break,
546
547                _ = self.probe_requests.notify.notified() => {
548                    if let Some(req) = self.probe_requests.take() {
549                        self.handle_request(req);
550                    }
551                }
552
553                Some(ev) = self.events.recv() => self.handle_event(ev),
554
555                Some(joined) = self.tasks.join_next() => {
556                    if let Err(err) = joined {
557                        // A task that panicked never sent its terminal
558                        // event, so account for it here and let the cycle
559                        // finalize. (Aborted tasks live in a dropped
560                        // JoinSet and are never yielded here.)
561                        if err.is_panic() {
562                            error!("probe task panicked: {err:#}");
563                        }
564                        self.probe_finished();
565                        self.advance();
566                    }
567                }
568
569                _ = &mut report_deadline => self.on_report_deadline(),
570                _ = &mut abort_deadline => self.on_abort_deadline(),
571            }
572        }
573    }
574
575    /// Handles a probe request: defer it, restart the cycle, or start one.
576    fn handle_request(&mut self, req: ProbeRequest) {
577        if self.relay_map.is_empty() {
578            debug!("skipping net_report, empty RelayMap");
579            return;
580        }
581
582        if let Some(cycle) = &mut self.cycle {
583            match req.scope {
584                ProbeScope::Refresh => {
585                    // Defer: run right after the current cycle finishes so
586                    // the trigger is not lost (the old DirectAddrUpdateState
587                    // remembered this via `want_update`). Only `Refresh`
588                    // requests are deferred, so the remembered scope stays
589                    // `Refresh`; just take the latest interface state.
590                    match &mut cycle.rerun {
591                        Some(pending) => pending.if_state = req.if_state,
592                        None => cycle.rerun = Some(req),
593                    }
594                    debug!("deferring probe request until current cycle finishes");
595                    return;
596                }
597                // Full: abort the current cycle and start fresh.
598                ProbeScope::Full => self.abort_cycle(),
599            }
600        }
601
602        self.start_cycle(req);
603    }
604
605    /// Aborts the current cycle's probe tasks and swaps in a fresh event
606    /// channel, so any results those tasks already queued are dropped instead
607    /// of leaking into the next cycle. Only a `Full` restart does this, and a
608    /// `Full` restart also closes the open QAD connections, so no observer
609    /// task is left running without an owner.
610    fn abort_cycle(&mut self) {
611        self.tasks = JoinSet::new();
612        let (tx, rx) = mpsc::channel(EVENT_CHANNEL_CAP);
613        self.events_tx = tx;
614        self.events = rx;
615        self.cycle = None;
616    }
617
618    /// Starts a new probe cycle.
619    fn start_cycle(&mut self, req: ProbeRequest) {
620        let ProbeRequest {
621            if_state,
622            scope: request_scope,
623        } = req;
624        let now = Instant::now();
625        let scope = self.reports.scope_for(request_scope, now);
626
627        debug!(?request_scope, ?scope, "starting probe cycle");
628
629        if scope == ProbeScope::Full {
630            #[cfg(not(wasm_browser))]
631            self.qad_conns.clear();
632            self.reports.last = None;
633            self.reports.next_full = false;
634            self.reports.last_full = now;
635            self.metrics.reports_full.inc();
636        }
637        self.metrics.reports.inc();
638
639        // Start the report from scratch. spawn_qad_probes copies the last
640        // address from each still-open QAD connection back in.
641        self.report = Report::default();
642        self.cancel_https = CancellationToken::new();
643
644        #[cfg(not(wasm_browser))]
645        let (qad_v4, qad_v6) = self.spawn_qad_probes(&if_state);
646        #[cfg(wasm_browser)]
647        let (qad_v4, qad_v6) = (0usize, 0usize);
648        let qad = QadState::new(qad_v4, qad_v6);
649        let mut pending = qad_v4 + qad_v6 + self.spawn_https_probes();
650        #[cfg(not(wasm_browser))]
651        if scope == ProbeScope::Full && self.user_config.captive_portal_check {
652            self.spawn_captive_portal();
653            pending += 1;
654        }
655
656        self.cycle = Some(ProbeCycle {
657            started: now,
658            pending,
659            qad,
660            published: false,
661            rerun: None,
662            report_deadline: Some(now + FIRST_REPORT_TIMEOUT),
663            abort_deadline: Some(now + ABORT_TIMEOUT),
664        });
665
666        // A cycle can start with no probes at all: every family already has
667        // an open connection and HTTPS is off. It is already complete, so
668        // finalize now to still update history and the preferred relay.
669        self.advance();
670    }

672    /// Applies one event to the report and drives the cycle forward.
673    fn handle_event(&mut self, ev: ProbeEvent) {
674        match ev {
675            #[cfg(not(wasm_browser))]
676            ProbeEvent::QadResult(family, run) => {
677                if let Some(c) = &mut self.cycle {
678                    c.qad.record_result(family, &run);
679                }
680                self.probe_finished();
681                match run {
682                    ProbeOutcome::Ok((report, conn)) => {
683                        debug!(?family, ?report, "QAD probe completed");
684                        // Accumulate: the first result sets the global
685                        // address; a second result from a different relay
686                        // decides mapping-varies-by-destination.
687                        self.report.apply_qad_result(family, &report);
688                        if self.qad_conns.slot(family).is_none() {
689                            // First result for this family: keep this
690                            // connection open, but let the other probes run
691                            // so a second result can decide mapping-varies.
692                            *self.qad_conns.slot_mut(family) = Some((report.relay_url, conn));
693                        } else {
694                            // Second result: mapping-varies is decided, so
695                            // stop the family's remaining probes and drop
696                            // this connection.
697                            conn.conn
698                                .close(QUIC_ADDR_DISC_CLOSE_CODE, QUIC_ADDR_DISC_CLOSE_REASON);
699                            self.qad_conns.cancel(family).cancel();
700                        }
701                        // UDP works, so skip captive portal detection.
702                        self.cancel_captive_portal.cancel();
703                    }
704                    ProbeOutcome::Err(e) => debug!(?family, "QAD probe failed: {e:#}"),
705                    ProbeOutcome::Timeout => debug!(?family, "QAD probe timed out"),
706                    ProbeOutcome::Cancelled => {}
707                }
708                self.publish();
709                self.advance();
710            }
711            #[cfg(not(wasm_browser))]
712            ProbeEvent::QadObservation(family, obs) => {
713                // Take observations only from the connection we kept for this
714                // family. A probe we dropped may still send one last
715                // observation before its task ends; ignore it.
716                let is_current = self
717                    .qad_conns
718                    .slot(family)
719                    .is_some_and(|(url, _)| *url == obs.relay_url);
720                if is_current {
721                    trace!(?family, ?obs, "QAD address observation");
722                    if let Some((_, conn)) = self.qad_conns.slot_mut(family) {
723                        conn.last = obs.clone();
724                    }
725                    self.report.apply_qad_observation(family, &obs);
726                    self.publish();
727                }
728            }
729            ProbeEvent::Https(run) => {
730                self.probe_finished();
731                match run {
732                    ProbeOutcome::Ok(report) => {
733                        debug!(?report, "HTTPS probe completed");
734                        self.report.apply_https_result(&report);
735                        if self.have_all_relay_latencies() {
736                            self.cancel_https.cancel();
737                        }
738                    }
739                    ProbeOutcome::Err(e) => debug!("HTTPS probe failed: {e:#}"),
740                    ProbeOutcome::Timeout => debug!("HTTPS probe timed out"),
741                    ProbeOutcome::Cancelled => {}
742                }
743                self.publish();
744                self.advance();
745            }
746            #[cfg(not(wasm_browser))]
747            ProbeEvent::CaptivePortal(run) => {
748                self.probe_finished();
749                match run {
750                    ProbeOutcome::Ok(found) => {
751                        debug!(found, "captive portal check completed");
752                        self.report.captive_portal = Some(found);
753                    }
754                    ProbeOutcome::Err(e) => debug!("captive portal check failed: {e:#}"),
755                    ProbeOutcome::Timeout => debug!("captive portal check timed out"),
756                    // Cancelled is expected: a successful QAD cancels this check.
757                    ProbeOutcome::Cancelled => {}
758                }
759                self.publish();
760                self.advance();
761            }
762        }
763    }

765    /// Records that one cycle probe has finished (however it exited).
766    fn probe_finished(&mut self) {
767        if let Some(c) = &mut self.cycle {
768            c.pending = c.pending.saturating_sub(1);
769        }
770    }

772    /// Publishes the current report, unless a guard holds it back.
773    ///
774    /// Two guards apply. An empty report is never published, since it would
775    /// overwrite addresses a caller has already seen. And the first report of
776    /// a cycle waits until every family the cycle probed has a result, so a
777    /// caller does not briefly see an IPv4-only report just before the IPv6
778    /// address arrives.
779    fn publish(&mut self) {
780        if !self.report.has_data() {
781            return;
782        }
783        if let Some(c) = &self.cycle
784            && !c.published
785            && !c.qad.all_determined()
786        {
787            return;
788        }
789        if let Some(c) = &mut self.cycle {
790            c.published = true;
791        }
792        self.report_out.set(Some(self.report.clone())).ok();
793    }
794
795    /// Finalizes the cycle if all its probes have finished.
796    fn advance(&mut self) {
797        if self.cycle.as_ref().is_some_and(|c| c.pending == 0) {
798            self.finish_cycle();
799        }
800    }

802    /// Commits the cycle to history, selects the preferred relay, and emits
803    /// the final report. Then applies any deferred rerun request.
804    fn finish_cycle(&mut self) {
805        let Some(cycle) = self.cycle.take() else {
806            return;
807        };
808        self.reports.record(&mut self.report);
809        // Emit the final report, now stamped with the preferred relay. This
810        // often repeats the last incremental value; `Watchable::set` only
811        // notifies watchers on change, so an identical final emit is a no-op.
812        // Keep the last good report rather than overwriting it with an empty
813        // one when a cycle discovered nothing.
814        if self.report.has_data() {
815            self.report_out.set(Some(self.report.clone())).ok();
816        }
817        debug!(
818            report = ?self.report,
819            duration = ?cycle.started.elapsed(),
820            "net_report cycle complete",
821        );
822        if let Some(rerun) = cycle.rerun {
823            self.handle_request(rerun);
824        }
825    }

827    /// The first-report deadline fired. Publish what we have now, even if
828    /// some probed families have not answered yet.
829    fn on_report_deadline(&mut self) {
830        debug!("report deadline fired");
831        if let Some(c) = &mut self.cycle {
832            c.report_deadline = None;
833            c.published = true;
834        }
835        self.publish();
836    }
837
838    /// Abort deadline: stop remaining probes and finalize with what we have.
839    fn on_abort_deadline(&mut self) {
840        debug!("abort deadline fired, finalizing cycle");
841        // Aborting drops the tasks, so they send no further events.
842        self.tasks = JoinSet::new();
843        if let Some(c) = &mut self.cycle {
844            c.abort_deadline = None;
845            c.pending = 0;
846        }
847        self.finish_cycle();
848    }

850    /// Returns `true` when every relay has at least one latency sample.
851    /// Used to cancel remaining HTTPS probes once coverage is complete.
852    fn have_all_relay_latencies(&self) -> bool {
853        let num_relays = self.relay_map.len();
854        if num_relays == 0 {
855            return true;
856        }
857        let mut seen = BTreeSet::new();
858        for (_, url, _) in self.report.relay_latency.iter() {
859            seen.insert(url);
860        }
861        seen.len() >= num_relays
862    }

864    /// Starts a QAD probe for each family that needs one, and returns how many
865    /// it started per family (v4, v6).
866    ///
867    /// A family needs a probe only when it has no open connection. Before
868    /// deciding, any connection that has since closed is dropped, and the
869    /// last address of each surviving connection is copied into the report.
870    #[cfg(not(wasm_browser))]
871    fn spawn_qad_probes(&mut self, if_state: &IfState) -> (usize, usize) {
872        let Some(quic_client) = self.quic_client.clone() else {
873            return (0, 0);
874        };
875
876        // Drop any connection that has closed, then copy the last address of
877        // each surviving one into the report.
878        for family in [AddrFamily::V4, AddrFamily::V6] {
879            if let Some((url, conn)) = self.qad_conns.slot(family)
880                && let Some(reason) = conn.conn.close_reason()
881            {
882                trace!(?family, ?url, "QAD conn closed: {reason}");
883                self.qad_conns.slot_mut(family).take();
884            }
885            if let Some(r) = self.qad_conns.current(family) {
886                self.report.apply_qad_observation(family, &r);
887            }
888        }
889
890        self.qad_conns.reset_cancels();
891
892        let families = [
893            (
894                AddrFamily::V4,
895                self.qad_conns.v4.is_none() && if_state.have_v4,
896            ),
897            (
898                AddrFamily::V6,
899                self.qad_conns.v6.is_none() && if_state.have_v6,
900            ),
901        ];
902
903        const MAX_RELAYS: usize = 5;
904        let mut spawned = 0;
905        let mut count_v4 = 0;
906        let mut count_v6 = 0;
907        for relay in self
908            .relay_map
909            .relays::<Vec<_>>()
910            .into_iter()
911            .take(MAX_RELAYS)
912        {
913            for (family, needed) in families {
914                if needed {
915                    // Stagger successive probes so they do not all fire at
916                    // once. The first probe (spawned == 0) starts immediately.
917                    let delay = self.user_config.qad_stagger * spawned as u32;
918                    self.spawn_qad_probe(family, relay.clone(), quic_client.clone(), delay);
919                    spawned += 1;
920                    match family {
921                        AddrFamily::V4 => count_v4 += 1,
922                        AddrFamily::V6 => count_v6 += 1,
923                    }
924                }
925            }
926        }
927        (count_v4, count_v6)
928    }

933        family: AddrFamily,
934        relay: Arc<iroh_relay::RelayConfig>,
935        quic_client: iroh_relay::quic::QuicClient,

949                self.events_tx.clone(),

955    /// Spawns HTTPS latency probes according to the current [`ProbePlan`].
956    /// Returns the number spawned.
957    fn spawn_https_probes(&mut self) -> usize {
958        let plan = match self.reports.last {
959            Some(ref report) => {
960                ProbePlan::with_last_report(&self.relay_map, report, &self.protocols)
961            }
962            None => ProbePlan::initial(&self.relay_map, &self.protocols),
963        };
964        trace!(%plan, "HTTPS probe plan");
965
966        let mut count = 0;
967        for probe_set in plan.iter() {
968            for (delay, relay) in probe_set.params() {
969                self.spawn_https_probe(*delay, Arc::clone(relay));
970                count += 1;
971            }
972        }
973        count
974    }

1019    /// Spawns a probe task into the actor's [`JoinSet`].
1020    ///
1021    /// The task waits out `delay`, runs `work` with a `timeout`, and can be
1022    /// aborted at any point through `cancel`. However the probe ends, the
1023    /// result becomes a [`ProbeOutcome`]; `to_event` maps it into a
1024    /// [`ProbeEvent`], which the task sends on the actor's channel. The delay
1025    /// sits outside the timeout, so a staggered probe still gets its full
1026    /// timeout once it starts.
1027    fn spawn_probe_task<T: MaybeSend, E: MaybeSend>(
1028        &mut self,
1029        span: Span,
1030        cancel: CancellationToken,
1031        delay: Duration,
1032        timeout: Duration,
1033        work: impl 'static + MaybeSend + Future<Output = Result<T, E>>,
1034        to_event: impl 'static + MaybeSend + FnOnce(ProbeOutcome<T, E>) -> ProbeEvent,
1035    ) {
1036        let events = self.events_tx.clone();
1037        self.tasks.spawn(
1038            async move {
1039                let outcome = cancel
1040                    .run_until_cancelled(async move {
1041                        if !delay.is_zero() {
1042                            time::sleep(delay).await;
1043                        }
1044                        time::timeout(timeout, work).await
1045                    })
1046                    .await;
1047                let outcome = match outcome {
1048                    Some(Ok(Ok(value))) => ProbeOutcome::Ok(value),
1049                    Some(Ok(Err(err))) => ProbeOutcome::Err(err),
1050                    Some(Err(_elapsed)) => ProbeOutcome::Timeout,
1051                    None => ProbeOutcome::Cancelled,
1052                };
1053                let event = to_event(outcome);
1054                events.send(event).await.ok();
1055            }
1056            .instrument(span),
1057        );
1058    }
1059}
